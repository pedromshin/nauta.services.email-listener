<objective>
Build the FastAPI write-side of region editing: seven use cases (accept / reject /
redraw / split / merge / nest / create-region) plus the endpoints, repository
methods, and DI registration that expose them. The Python service owns supersede
semantics (D-16) and tenancy (D-18). Everything must work with ZERO proposed
regions — the create-region endpoint is the "Add region" backend.

Purpose: Region edits are domain writes. They live in FastAPI behind X-API-Key so
the browser never holds the key. Supersede-not-mutate keeps an auditable lineage.

Output: edit_region.py use cases, extended component repository + port, seven new
endpoints in components.py, container wiring, full Python test coverage, and one
small env-gated round-trip in the real-Postgres integration test.
</objective>
