<objective>
Build the typed write-path the browser uses without ever touching the API key: two
pure geometry helpers (pointer→normalized, rect→polygon) with vitest coverage, and
seven server-side tRPC mutations that proxy to the FastAPI region-edit endpoints
using EMAIL_LISTENER_URL + EMAIL_LISTENER_API_KEY. Plus the .env.example plumbing.

Purpose: tRPC mutations are the only bridge from apps/web to the write endpoints.
Running server-side keeps EMAIL_LISTENER_API_KEY out of the client bundle. The
geometry helpers are the inverse of polygonToRect, needed by draw mode.

Output: extended geometry.ts + tests, mutations.ts (new), emailsRouter wiring,
.env.example entries. No FastAPI dependency at build time (env read at call time).
</objective>

    -> ReadonlyArray<readonly [number, number]>   // [ [l,t],[r,t],[r,b],[l,b] ]

  ...emailDetailProcedures, list, byId }). Spread componentMutationProcedures the
same way detail procedures are spread. publicProcedure is imported from "../../trpc".

Server-side env-guard analog: apps/web/src/app/api/attachments/[id]/route.ts reads
process.env at call time and throws "X is not configured" when missing — replicate
as getListenerConfig() so Next build does not fail when env is absent at build.

FastAPI endpoint contract (from Plan 06-01, same phase):
  POST {url}/v1/components/{id}/accept            body {}
  POST {url}/v1/components/{id}/reject            body {}
  POST {url}/v1/components/{id}/redraw            body {polygon, page_index}
  POST {url}/v1/components/{id}/split             body {regions:[{polygon,page_index}]}
  POST {url}/v1/components/merge                  body {component_ids, polygon?, page_index?}
  POST {url}/v1/components/{id}/nest              body {parent_component_id|null}
  POST {url}/v1/components/{pageComponentId}/regions  body {polygon, page_index}
  Header on every call: "X-API-Key": <key>, "Content-Type": "application/json".
  Non-2xx → parse {detail} and throw Error(detail ?? "<op> failed").
