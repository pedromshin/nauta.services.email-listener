<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser DOM event → mutation input | Pointer coords + selected ids become mutation arguments |
| client → tRPC mutation (Plan 06-02) | The only path to the write endpoints; key stays server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-11 | Tampering | pointer-derived polygon | mitigate | clientXYToNormalized clamps to [0,1]; min-size 0.01 guard; tRPC zod (06-02) + Pydantic (06-01) re-validate before persistence — client is never trusted |
| T-06-12 | Information disclosure | mutation error surface | mitigate | onError shows a friendly toast ("Could not <op> region. Try again."); the raw error is only console-logged, never rendered |
| T-06-13 | Tampering | optimistic cache mutation | mitigate | setData uses immutable spreads + a captured prev snapshot reverted on error; no in-place mutation of query data |
| T-06-14 | Spoofing | API key in client bundle | mitigate | No key referenced in apps/web; all writes go through server-side tRPC (06-02); grep gate in 06-02 enforces no NEXT_PUBLIC_ key |
| T-06-SC | Tampering | pnpm installs | mitigate | No new packages (sonner is transitive via @nauta/ui/sonner, already present); if any appear, slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] |
</threat_model>
