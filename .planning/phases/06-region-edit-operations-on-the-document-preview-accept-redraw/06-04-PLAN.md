<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser selection state → merge/nest mutation | Selected ids + parent id become mutation arguments |
| client → tRPC mutation (Plan 06-02) | Server-side proxy holds the key; FastAPI re-validates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-15 | Elevation/IDOR | merge across emails/attachments | mitigate | UI only offers same-page regions for nest; merge eligibility is client-side convenience — FastAPI (06-01) is the enforcement point, raising 404 when component_ids do not share email+attachment |
| T-06-16 | Tampering | nest into ineligible parent | mitigate | nest-picker filters to same-page, non-rejected/superseded, non-self; tRPC zod validates uuid/null; Pydantic re-validates |
| T-06-17 | Repudiation | destructive reject without confirm | mitigate | AlertDialog confirmation gates reject; reject is a status write (rejected), reversible by re-accepting/redraw — lineage stays in content_raw |
| T-06-18 | Information disclosure | history view leaks superseded data | accept | Superseded/rejected regions belong to the same email/tenant; "Show history" only reveals the user's own email's lineage (no cross-tenant exposure; tenancy is per-email read, D-18) |
| T-06-SC | Tampering | pnpm installs | mitigate | No new packages (alert-dialog/popover/checkbox already in @nauta/ui); if any appear, slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] |
</threat_model>
