<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inbox row data -> rendered HTML | Email subject/sender/body and entity-type labels are untrusted content rendered in the list/preview. |
| chip href construction | emailId placed into a /emails/{id} link. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-50 | Tampering (XSS) | inbox row/preview rendering email content | mitigate | Subject/sender/labels rendered as React text nodes (auto-escaped); the reading-preview body snippet is plain text (no dangerouslySetInnerHTML — full HTML body sanitization stays in the existing /emails/[id] DOMPurify path, not the inbox snippet). |
| T-09-51 | Tampering | chip href injection | mitigate | emailId is a server-provided UUID from emails.list; rendered via next/link `href={/emails/${emailId}}` (no user-controlled path segment). |
| T-09-52 | Information Disclosure | over-fetching entity data per row | mitigate | Single batched `entitySummary` query keyed by the visible page; capped at the inbox page size; no per-row fetch (D-23). |
| T-09-53 | Information Disclosure | API key in client bundle | accept | This plan is presentation-only; all data comes via tRPC queries (no fetch to FastAPI, no key) — writes are not introduced here. |
| T-09-SC | Tampering | npm/pip/cargo installs | mitigate | No new npm packages (sidebar/resizable/next-themes/lucide already present via 09-05/@nauta/ui); legitimacy gate N/A. |
</threat_model>
