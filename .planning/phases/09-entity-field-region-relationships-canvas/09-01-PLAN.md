<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| schema → live DB | Drizzle TS types derive from schema files, NOT the live DB; a schema edit without a migration apply yields code that typechecks but queries a column that does not exist in Postgres. |
| component FK → entity_types/entity_type_fields | New nullable FKs; a deleted entity-type/field must not orphan or cascade-delete components. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | migration drift (schema edited, not applied) | mitigate | [BLOCKING] Task 2 generates AND applies the migration to local Postgres and verifies the columns via information_schema; staging/prod recorded as deploy follow-up. |
| T-09-02 | Denial of Service | FK cascade on entity-type delete | mitigate | `entity_type_id` / `entity_type_field_id` use `onDelete: "set null"` (D-03/D-04) — deleting a referenced entity-type/field nulls the component link rather than cascade-deleting components; hard deletes are further guarded in 09-03 (D-27 delete-guard). |
| T-09-03 | Information Disclosure | enum/column leakage | accept | role/entity_type ids are non-sensitive structural metadata; no PII added. |
| T-09-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed (drizzle-kit/tsx already in devDependencies); slopcheck N/A. |
</threat_model>
