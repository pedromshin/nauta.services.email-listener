## Layout Shell (D-06 — LOCKED)

Phase 9 replaces the two-panel `/emails/[id]` layout with a four-zone editor shell. The shell occupies the full viewport height below the application header (if any) and uses `flex flex-row` with no padding wrapper.

### Panel Dimensions

| Zone | Width | Class | Notes |
|------|-------|-------|-------|
| LAYERS (left) | 256px fixed | `w-64 shrink-0` | Entity/field tree; scrollable via ScrollArea |
| CANVAS (center) | `flex-1 min-w-0` | `flex-1 min-w-0` | PDF + overlay; overflow-hidden; contains its own scroll/pan area |
| INSPECTOR (right) | 288px fixed | `w-72 shrink-0` | Role picker + relationship pickers; scrollable |
| TOOLBAR (top) | 100% width, 44px height | `h-11 flex items-center` | Full-width bar above the three columns |

## Color

Inherited from Phase 5 token contract. Phase 9 introduces a **role-color palette** for the four region roles.

### Base Palette (60/30/10 split)

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `bg-background` | white (hsl 0 0% 100%) | Canvas background, panel backgrounds |
| Secondary (30%) | `bg-muted` / `bg-card` | hsl 0 0% 96.1% | LAYERS panel bg, INSPECTOR panel bg, toolbar bg, tree row hover |
| Accent (10%) | `bg-primary` | hsl 164 39% 22% (dark teal-green) | Active-parent ENTITY box ring, confirm buttons, confirmed badge, selected tree node indicator |
| Destructive | `bg-destructive` | hsl 0 84% 60% | Deny (✗) button, low-confidence score text, deny confirmation copy |

Accent reserved for:
- Active-parent entity box border + ring on canvas
- Confirmed field badge background
- "Confirm fields" primary action button
- Selected LAYERS tree node left-edge indicator (3px border-l)
- Active (armed) tool-mode button in toolbar — segmented-control indicator `bg-primary/10 text-primary border border-primary/30` on the active Select or Draw button (the inactive button stays ghost/`text-muted-foreground`)

Source: Phase 5 UI-SPEC §Color; Phase 6 UI-SPEC overlay colors; Phase 7 UI-SPEC confirmed badge.

### Role-Color Palette (NEW — Phase 9)

Each `component_role` value maps to a distinct box color on the canvas and a matching chip in the LAYERS tree. Colors are chosen to be accessible at AA contrast on white, visually distinct from each other, and not clashing with the primary teal-green accent.

| Role | Overlay Border | Overlay Fill | Label Chip | Tree Icon |
|------|---------------|--------------|------------|-----------|
| `entity` | `border-violet-500` | `bg-violet-500/10` | `bg-violet-100 text-violet-800` | Square icon, violet |
| `field` | `border-amber-500` | `bg-amber-500/10` | `bg-amber-100 text-amber-800` | Dot icon, amber |
| `unrelated` | `border-slate-400` | `bg-slate-400/8` | `bg-slate-100 text-slate-600` | X icon, slate |
| `unclassified` (null) | `border-primary/60` | `bg-primary/10` | — (no chip) | Inherit existing overlay color |

When a role is set, the existing `border-primary/80 bg-primary/10` base overlay class is replaced with the role-specific class. Selected state adds `ring-2 ring-{role-color}/50` on top. Mutating adds `animate-pulse opacity-70`.

Active-parent ENTITY box (the currently selected entity when D-10 active-parent model is engaged) adds an additional outer ring: `ring-4 ring-violet-400/40` to visually signal "next drawn boxes become fields of this entity."

### Inline ✓/✗ Overlay Controls (D-16/D-17/D-18)

Confirm button: `bg-green-500 hover:bg-green-600 text-white rounded-full h-5 w-5 flex items-center justify-center text-xs font-bold`
Deny button: `bg-destructive hover:bg-destructive/90 text-white rounded-full h-5 w-5 flex items-center justify-center text-xs font-bold`
Position: `absolute -top-3 right-0 flex gap-1 z-30` — floats above the top-right corner of the box, pointer-events-auto.

These controls are rendered on boxes that have a `candidate_value` (i.e., autofill-generated FIELD boxes with a pending candidate value). Confirmed boxes show no inline controls.

### Zoom Group (center, after separator)

| Control | Icon / Label | Aria-label | Keyboard |
|---------|-------------|------------|----------|
| Zoom out | `ZoomOut` (Lucide) | "Zoom out" | `Cmd/Ctrl + Minus` |
| Zoom % display | text "{N}%" (button) | "Current zoom, click to reset to 100%" | `Cmd/Ctrl + 0` = reset to 100% |
| Zoom in | `ZoomIn` (Lucide) | "Zoom in" | `Cmd/Ctrl + Equals` |
| Fit width | "Fit width" text button | "Fit page to panel width" | `Cmd/Ctrl + Shift + W` |

## LAYERS Panel (left, 256px — D-06, D-12)

The LAYERS panel shows the entity→field tree. Header: `"Regions"` (text-sm font-semibold px-3 py-2 border-b bg-background).

### Tree Structure

```
[entity row]                        ← role=entity, always visible
  [field row 1]                     ← role=field, child of entity; visible when parent entity selected (D-12)
  [field row 2]
[entity row 2]
[unclassified row]                  ← role=null; always visible
[unrelated row]                     ← role=unrelated; hidden unless Unrelated toggle ON (D-12, D-05)
```

### Tree Row Anatomy

Each row is `role="treeitem"` inside `role="tree"`. Rows are 36px tall (`py-2 px-3`).

ENTITY row:
```
[▸/▾ chevron (ChevronRight/ChevronDown, h-3 w-3)] [role chip: violet] [label text-sm font-semibold truncate] [page badge text-xs text-muted-foreground ml-auto]
```
- Chevron toggles FIELD children visibility (D-12: field rows collapse when entity deselected)
- Click = select entity, sets active-parent (D-10)
- Active-parent entity row: `bg-primary/10 border-l-2 border-primary`

FIELD row (indented 20px left padding, `pl-8`):
```
[role chip: amber text-xs] [property label text-sm font-semibold truncate] [:] [candidate value text-sm font-normal text-muted-foreground truncate] [inline ✓ h-4 w-4] [inline ✗ h-4 w-4]
```
- Visible only when parent entity is expanded (D-12)
- Inline ✓/✗ inline within the row (not as floating overlay — LAYERS rows are text-only; the canvas overlay gets the floating controls)
- Confirmed field row: shows `bg-green-50` background, no ✓/✗ buttons

UNCLASSIFIED row (role=null, no chip):
```
[square-dashed icon h-3 w-3 text-muted-foreground] [label/status text-sm text-muted-foreground truncate]
```

### Empty State

When no regions exist:
```
Header: "No regions yet"
Body:   "Draw a rectangle on the document to define your first region."
```
Centered in the panel, `py-12 px-4 text-center text-sm text-muted-foreground`.

### Role Chips

Inline role chips in the tree (not the canvas boxes) are compact: `text-xs px-2 py-1 rounded-sm font-semibold`.

| Role | Classes |
|------|---------|
| entity | `bg-violet-100 text-violet-800` |

### Inline ✓/✗ Controls (D-16, D-17, D-18)

Shown on FIELD boxes that have a candidate value and are not yet confirmed. Position: `absolute -top-3 right-0 flex gap-1 z-30 pointer-events-auto`.

```tsx
<div className="absolute -top-3 right-0 flex gap-1 z-30">
  <button
    aria-label="Confirm field value"
    className="h-5 w-5 rounded-full bg-green-500 hover:bg-green-600 text-white flex items-center justify-center text-xs font-bold"
    onClick={onConfirm}
  >
    ✓
  </button>
  <button
    aria-label="Deny field value"
    className="h-5 w-5 rounded-full bg-destructive hover:bg-destructive/90 text-white flex items-center justify-center text-xs font-bold"
    onClick={onDeny}
  >
    ✗
  </button>
</div>
```

Confirm (✓) → promotes candidate to confirmed; confirmed boxes display `bg-green-500` border variant and remove ✓/✗ controls.

Deny (✗) origin-aware (D-18):
- Auto-detected origin: removes box from view (soft-reject, status → `rejected`)
- User-drawn origin: keeps box geometry, clears candidate value + entity_type_field_id only; box reverts to unclassified state

Per D-19: once an auto-detected box is denied, its canonical polygon is stored in a rejection memo so it is not re-proposed on re-run.

### Drawing Model (D-08, D-09)

Default drag on empty canvas = draw rectangle. No mode switch required. `DrawOverlay` component (existing) is always mounted in Phase 9 (not only when draw mode is active); however, it must not intercept clicks on existing boxes — only pointer-down on the canvas background triggers draw.

Implementation note: existing `draw-overlay.tsx` is rendered with `z-20 cursor-crosshair`. Phase 9 must conditionally render it or use `pointer-events-none` when hovering over an existing box to allow box selection.

Select mode (toolbar): pointer-down on empty area does nothing; drag selects by rubber-band (optional future enhancement — not in Phase 9 scope).

Draw mode (toolbar): identical to existing behavior — `cursor-crosshair`, drag creates rectangle.

Esc key: cancels in-progress draw AND deselects selection (existing behavior preserved).

Move/resize semantics (D-09): dragging an existing box corner or edge creates a new `email_components` row (supersedes old), never in-place mutation. This is the existing "redraw" semantic from Phase 6.

Min draw size: `MIN_DRAW_SIZE = 0.01` (1% of page in each dimension) — inherited from `draw-overlay.tsx`. Too-small draw fires `toast.warning("Rectangle too small. Draw a larger area.")`.

  - Rows: field label + key (text-sm / text-xs muted)
  - Only enabled after parent entity is selected

Optimistic update: same pattern as role — optimistic local then mutation, revert on error.

#### Section 4: Sub-field Autofill (D-13, D-14, D-15)

Button: `variant="default" size="sm" w-full` — "Autofill Fields"

- Default: "Autofill Fields" (enabled)
- Extracting: `Loader2 animate-spin h-4 w-4` inline + "Extracting…" (disabled)
- Done: auto-reverts to default; results appear in LAYERS + canvas as candidate FIELD boxes
- Failed (Bedrock degraded): toast `"AI autofill is unavailable — model access is pending."` (6000ms); button reverts to default

"Confirm all" affordance (Discretion): After autofill produces candidate FIELD boxes, a secondary `variant="outline" size="sm" w-full` button "Confirm All Fields" appears below the primary button. This bulk-confirms all candidate fields of the selected entity in one click. Copy: `"Confirm All Fields"`. Tooltip: "Confirms all candidate field values for this entity. You can still edit them individually afterward."

Per D-14: nothing is auto-confirmed; "Confirm All" is a conscious user action.

#### Section 5: Candidate Value (visible when role = field AND candidate_value present)

Label: `"Candidate value"` (text-xs font-semibold uppercase text-muted-foreground)

Inline edit input: `Input h-8 text-sm` (same as FieldsPanel pattern).

Confidence threshold display: ≥0.5 = `text-muted-foreground`; <0.5 = `text-destructive` per Phase 7 pattern.

## Active-Parent Model (D-10)

When a REGION with `role = entity` is selected in the canvas or LAYERS panel:
- That entity becomes the "active parent"
- Any subsequently drawn rectangle is automatically assigned `role = field` and `parent_component_id` = the active entity's ID
- The active entity box gains the `ring-4 ring-violet-400/40` outer ring (see Canvas section)
- The toolbar displays a contextual banner (same `DrawModeBar` pattern): `"Active entity: {entity label} — next drawn boxes become fields"` with a "Clear" ghost button (Esc also clears)
- When active entity is deselected (click empty canvas, Esc, or explicit Clear): drawn boxes revert to unclassified

## Keybindings Contract

| Key | Action | Condition |
|-----|--------|-----------|
| `V` or `S` | Activate Select tool | Always |
| `D` | Activate Draw tool | Always |
| `Escape` | Cancel in-progress draw; deselect selection; clear active-parent | — |
| `Delete` | Open reject dialog for selected region | Single region selected, not terminal |

## API + Mutation Patterns (D-15)

All writes route through the FastAPI proxy. Browser never holds `X-API-Key`.

| Mutation | Endpoint | Trigger | Optimistic? |
|----------|----------|---------|-------------|
| Set component role | `PATCH /v1/components/{id}/role` | Role picker change | Yes — revert on error |
| Set entity_type_id | `PATCH /v1/components/{id}/entity-type` | Entity type picker select | Yes — revert on error |
| Set field relationship (parent + property) | `PATCH /v1/components/{id}/field-relationship` | Parent picker + field picker | Yes — revert on error |
| Autofill fields | `POST /v1/components/{entity_id}/autofill-fields` | "Autofill Fields" button | No — async, results appear as new candidates |
| Confirm field candidate | `POST /v1/components/{field_id}/confirm` | ✓ click | Optimistic — show confirmed state immediately |
| Deny field candidate | `POST /v1/components/{field_id}/deny` | ✗ click | Optimistic — remove/clear immediately, undo on error |

tRPC router mutations: `components.setRole`, `components.setEntityType`, `components.setFieldRelationship`, `components.autofillFields`, `components.confirmField`, `components.denyField`.

tRPC queries: `entityTypes.list` (existing from Phase 7), `components.listEntityRegions` (new — lists entity-role regions on a given page for parent picker).

Supersede-never-mutate (D-09): geometry changes (move, resize = redraw) create new `email_components` row, mark old as `superseded`. This is the existing Phase 6 pattern; Phase 9 extends it to include role + relationship columns on the new row.

Tenant isolation: `importer_id` is derived from the `email_components` row's lineage (via attachment → email → importer), never from the caller. This is the existing Phase 4 pattern.
