<objective>
Ship the human curation loop the suggest-only model requires (D-20): confirm-merge, reject-merge, and
unmerge use cases + FastAPI endpoints. Confirm-merge is the act that actually links two identities (and
fires the D-11 flywheel write-back); reject dismisses a suggestion durably; unmerge undoes a wrong
confirmed merge (silent-corruption cost means undo must exist in-product). These are the writes the
10-04 tRPC mutations proxy and the 10-06 detail page drives.

requires confirm/reject/unmerge to exist server-side. This plan is serialized after 10-02 because it
extends the same shared files (entity_instances.py router, container.py, entity_instance_repository.py).
Output: a curation use-case module, the repo methods + endpoints + DI for the three actions, and a
fake-repo test suite.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /v1/entity-instances merge/unmerge | untrusted UUID path params for two identities cross here |
| curation use case → entity_instances / candidate_links | a merge across tenants would corrupt another importer's shipment record |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-20 | Information disclosure / Tampering | ConfirmMerge / RejectMerge / Unmerge | mitigate | importer_id derived from the loaded entity row (D-21); both ids validated to share the importer before any write — a cross-tenant merge is HIGH-severity and is rejected as ValueError->404; every repo query stays `.eq("source","email_extracted")` |
| T-10-21 | Spoofing | merge/unmerge endpoints | mitigate | router-level `Depends(require_api_key)` (X-API-Key); browser never holds the key — calls arrive only via the 10-04 tRPC server-side proxy |
| T-10-22 | Repudiation | merge decision | mitigate | confirm sets was_selected=true on the candidate link (D-09 audit trail) so every human merge decision is recorded; unmerge is supersede-not-destroy so the prior state is auditable |
| T-10-23 | Tampering | malformed ids | mitigate | UUID path params -> Pydantic 422; no string interpolation into SQL (Supabase client parameterizes) |
| T-10-SC | Tampering | npm/pip installs | mitigate | No new packages installed; no Package Legitimacy Gate triggered |
</threat_model>
