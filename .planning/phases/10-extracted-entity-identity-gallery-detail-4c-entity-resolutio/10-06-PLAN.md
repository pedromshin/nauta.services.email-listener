## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → tRPC mutations | confirm/reject/unmerge originate in the client |
| tRPC server → FastAPI | X-API-Key attached server-side; must never reach the client bundle |
| entity-derived strings → DOM | display names, identifiers, field values, knowledge content rendered in the detail page |
| emails.entitySummary → entity_instances | a new cross-table read resolving each entity region to its identity for the chip deep-link |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-60 | Spoofing / Information disclosure | EMAIL_LISTENER_API_KEY | mitigate | curation writes go through `api.entities.confirmMerge/rejectMerge/unmerge` (10-04 server-side proxy); no component holds the key, no fetch/X-API-Key under apps/web/src/app/entities/[id]/ (grep gate); per D-21 |
| T-10-61 | Cross-site scripting | display name / identifiers / field values / knowledge content / email subjects | mitigate | all entity-derived strings rendered as React text children (auto-escaped); NO `dangerouslySetInnerHTML` in any detail component (follows the DOMPurify/escaping convention) |
| T-10-62 | Tampering | optimistic curation updates | mitigate | optimistic state is a client-only view; the server is the source of truth — onError reverts via the snapshot and onSuccess invalidates from the server (no client-trusted write) |
| T-10-63 | Information disclosure | byId tenant scope | mitigate | the page reads only `api.entities.byId` (10-04), which is source='email_extracted' scoped server-side; the client cannot widen scope |

| T-10-SC | Tampering | npm installs | mitigate | No new npm packages — card/badge/button/skeleton/alert-dialog from @nauta/ui + lucide + sonner already installed (UI-SPEC Registry Safety); no Package Legitimacy Gate triggered |
