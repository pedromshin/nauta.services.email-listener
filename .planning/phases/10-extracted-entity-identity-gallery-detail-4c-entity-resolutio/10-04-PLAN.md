<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → tRPC (server) | inputs (ids, search term, pagination) cross from client to server |
| tRPC server → FastAPI | X-API-Key is attached server-side; must never reach the client bundle |
| tRPC query → Postgres | search term + filters reach SQL; must be parameterized |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-30 | Information disclosure | EMAIL_LISTENER_API_KEY | mitigate | key read only inside getListenerConfig() at request time (server), never NEXT_PUBLIC_, never in client-importable code (verified by grep gate in Task 3); proxied through FastAPI per D-21 |
| T-10-31 | Information disclosure | list / byId tenant scoping | mitigate | every entity_instances query is `source='email_extracted'` + importer-scoped; nauta_sync rows never surfaced; byId resolves by id but only exposes email-extracted data |
| T-10-32 | Injection | pg_trgm search term | mitigate | the search term is a bound parameter in a Drizzle `sql` fragment (similarity()/ILIKE), never string-interpolated (T-05-03); all filters use parameterized eq()/and() builders |
| T-10-33 | Denial of service | list pagination / search | mitigate | limit capped at .max(100), search capped at .max(200), emailIds-style bounded inputs; limit+1 idiom avoids unbounded scans |
| T-10-34 | Tampering | status/sort inputs | mitigate | z.enum allow-lists for status and sort reject any out-of-set value at the tRPC boundary |
| T-10-SC | Tampering | npm installs | mitigate | No new npm packages — drizzle-orm, zod, @nauta/db already present; no Package Legitimacy Gate triggered |
</threat_model>
