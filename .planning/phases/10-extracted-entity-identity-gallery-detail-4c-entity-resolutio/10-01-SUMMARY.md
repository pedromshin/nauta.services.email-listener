## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] IMMUTABLE wrapper required for aliases GIN expression index**
- **Found during:** Task 2 (applying 0017 to local Postgres)
- **Issue:** `CREATE INDEX ... USING gin ((coalesce(array_to_string(aliases, ' '), '')) gin_trgm_ops)` raised error 42P17: "functions in index expression must be marked IMMUTABLE". Both `array_to_string` and `coalesce` are STABLE (not IMMUTABLE) in the local Supabase Postgres build. Two inline attempts failed.
- **Fix:** Created `immutable_array_to_text(arr text[]) RETURNS text LANGUAGE sql IMMUTABLE PARALLEL SAFE` wrapper function in 0017 before the GIN index DDL. Index now references `immutable_array_to_text(aliases)`. The function also handles NULL via coalesce internally.
- **Files modified:** packages/db/migrations/0017_entity_resolution_rpcs.sql
- **Commit:** 4cb0c6f

**2. [Rule 3 - Blocking] Partial unique index applied manually to local Postgres**
- **Found during:** Task 3 (verifying live DB shape)
- **Issue:** drizzle-kit `generate` created `0016_careless_shen.sql` (correct scoped content) which the migrator applied as idx=16 entry. When the file was renamed to `0016_entity_identity.sql` with the partial unique index DDL added (the scoped file had the correct content already including the partial unique), the `__drizzle_migrations` table already had the original entry recorded. Local DB was missing the partial unique index `entity_instance_unique_per_importer`.
- **Fix:** Applied partial unique index directly via a tsx verify script that ran `CREATE UNIQUE INDEX IF NOT EXISTS "entity_instance_unique_per_importer" ON "entity_instances" ("importer_id","entity_type_id","nauta_id") WHERE "nauta_id" IS NOT NULL`. Script deleted after use. For staging/prod, 0016_entity_identity.sql will apply correctly since those DBs have no prior 0016 entry.
- **Files modified:** none (live-only fix; 0016_entity_identity.sql already contained the correct DDL)
- **Commit:** d95bc11 (confirmed the index was present before this commit)
