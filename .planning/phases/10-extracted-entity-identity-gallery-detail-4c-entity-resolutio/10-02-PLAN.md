<objective>
Build the defensible resolution core (D-05..D-13): a populated `entity_instances` identity store, a
parallel BlendedRAG resolver (dense HNSW + lexical pg_trgm fused by RRF k=60), per-arm provenance into
`component_entity_candidate_links`, on-confirm incremental promotion + resolution, and a re-runnable
backfill command. Reader endpoints (`GET /candidates`, `POST /backfill`) live here; the human
confirm/reject/unmerge CURATION endpoints are 10-03.

Purpose: this is the "4c entity resolution" deliverable and a design-case defense artifact — logistics
identifiers REQUIRE the lexical arm (MSCU1234567 vs MSCU1234568 are ~0 apart to an embedding), so the
system fuses two weak signals via RRF and always routes to a human (D-05 suggest-only). Bedrock may
404, so every path degrades to lexical-only (D-12). The corpus is empty today, so a backfill that
(re)builds it is essential (D-10).
Output: domain entity + port + repo for entity_instances, the resolution repo, resolve/promote/backfill
use cases, two read/command endpoints, DI + router wiring, and a fake-repo test suite.
</objective>
