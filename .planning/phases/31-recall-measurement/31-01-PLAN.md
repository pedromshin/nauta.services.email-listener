<objective>
Close the verified few-shot rendering gap and inject a resolved entity's already-known
`aliases[]` and `identifiers` into the autofill prompt as a delimited "known entity context"
block (RECALL-01). Cheap recall: a direct `entity_instances` read via the existing suggest-only
link paths — NO BFS, NO graph traversal, ZERO new migrations.

Purpose: The RRF retrieval pipeline already produces top-3 examples, but `_generate` never
renders them into the Bedrock messages, so retrieved context never reaches the model. This plan
makes both the retrieved examples AND the resolved entity's aliases/identifiers observable in the
constructed prompt.
Output: Updated adapter (few-shot + entity-context rendering), extended `AutofillProtocol`,
use-case wiring that reads the resolved entity best-effort, and container DI.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| entity_instances data -> LLM prompt | Aliases/identifiers are stored content, treated as UNTRUSTED input crossing into the model prompt |
| component.importer_id -> entity read | Tenant boundary: only the component's own importer's entity may be injected |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering (prompt injection) | autofill_adapter `_generate` | mitigate | Render aliases/identifiers + examples ONLY in the user turn inside dedicated delimiters (`<known_entity_context>`, `<example>`); NEVER in the system prompt — preserves D-14 structural defense |
| T-31-02 | Information Disclosure (cross-tenant) | autofill.py entity read | mitigate | Resolve the entity via component-scoped link paths using the D-18 importer derivation; assert no other importer's instance is injected (tenant-scoping test) |
| T-31-03 | Denial of Service (prompt bloat) | entity_context rendering | mitigate | Defensive alias cap (module constant) bounds the injected block size |
| T-31-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
