<objective>
Surface the forwarding seam to the user (THRD-04, web half): a `forwarding` tRPC router whose `getOrCreateMyAddress` procedure generates a CSPRNG secret token for the caller (idempotent, one per user), persists it to `forwarding_addresses`, and returns the full `u-{token}@{FORWARDING_EMAIL_DOMAIN}` address; a minimal web surface that shows the address with a copy affordance; and a user onboarding runbook covering the SES domain catch-all routing (user-gated terraform apply, deferred) and Gmail's destination-verification handshake.

Purpose: The seam is only useful if the user can see their address and knows how to point Gmail at it. Token generation lives here (web/session side) because users live in the web app; the FastAPI side (Plan 45-05) only RESOLVES tokens. Scope is minimal surfacing + runbook — full onboarding UX is deferred (CONTEXT).
Output: forwarding router + registration, a forwarding-address card + a small settings page, and FORWARDING-RUNBOOK.md.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web client → forwarding tRPC | userId must come from ctx.user, never client input; token never client-supplied |
| token generation → secrecy | a low-entropy or predictable token undermines the whole seam |
| forwarding address → display | the address is a semi-secret; must not be logged or leaked to other users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-06-01 | Spoofing | one user obtaining/guessing another's forwarding token | mitigate | getOrCreateMyAddress keys strictly on ctx.user.id (protectedProcedure); token is CSPRNG ≥128-bit; UNIQUE(userId) means one address per user; RLS (Plan 45-01) denies cross-user reads on the non-superuser path |
| T-45-06-02 | Information Disclosure | token/address leaking via logs or error messages | mitigate | never log the address; friendly client errors, detailed errors server-side only (repo guardrail) |
| T-45-06-03 | Tampering | missing FORWARDING_EMAIL_DOMAIN yields a blank/invalid address | mitigate | env validated fail-fast at boundary (surface at startup, repo guardrail) |
| T-45-06-SC | Tampering | npm install for token generation/UI | accept | Uses Node built-in crypto + existing @polytoken/ui components — no new package installs |
</threat_model>
