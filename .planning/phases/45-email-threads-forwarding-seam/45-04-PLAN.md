<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web client → emails tRPC | client-supplied importerId/pagination is untrusted; scope derives from session |
| thread grouping query → tenant isolation | a thread entry must never include another user's emails |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-04-01 | Information Disclosure | thread-grouped query leaking cross-tenant threads | mitigate | reuse userOwnedImporterIds + resolveListScope verbatim; grouping filters `inArray(importerId, scope.importerIds)`; test asserts user B cannot see user A's threads |
| T-45-04-02 | Denial of Service | unbounded memberEmailIds payload on a huge thread | mitigate | cap members returned per thread (e.g. most-recent 50); paging preserved on the group list |
| T-45-04-03 | Elevation of Privilege | client-supplied importerId narrowing to an unowned importer | mitigate | resolveListScope returns {ok:false} → empty result for any unowned/absent importerId |
| T-45-04-SC | Tampering | npm install for UI | accept | Reuse existing @polytoken/ui components (Badge/Button/Skeleton/Resizable already vendored); no new package installs |
</threat_model>
