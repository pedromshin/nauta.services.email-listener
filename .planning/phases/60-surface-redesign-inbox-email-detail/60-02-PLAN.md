§C — Data actually available on the row. Read this before designing; it bounds the work:

  `InboxThreePane` resolves members from `emails.list`, whose row type is
  `InboxEmailItem extends InboxEmail { bodyText: string | null; toAddresses: ReadonlyArray<string> }`.
  `InboxThreadGroup` types `members` as `InboxEmail` — the NARROWER type — so `bodyText` is present
  at runtime but invisible to the type. Widening `InboxEmail` with `bodyText` is what unlocks the
  per-message snippet; the data already flows.

      ink dot is therefore NOT implementable in this phase. Do not add one. Do not add a fake one.
      The dead filter is pre-existing and out of scope — note it in the summary for the backlog.

      reference's `.attach` clip row is not implementable here. Do not add it.
