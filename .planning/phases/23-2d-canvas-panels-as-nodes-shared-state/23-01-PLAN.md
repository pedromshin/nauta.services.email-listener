<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web client → tRPC saveCanvasLayout | Untrusted canvas snapshot (positions, edges, shared_state) crosses into persisted jsonb |
| persisted jsonb → later reads | A tampered/legacy row is read back on canvas mount |
| anon/authenticated Postgres roles → chat_canvas_layouts | RLS boundary (backend service-role only) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering | saveCanvasLayout edge/shared_state payload | mitigate | CanvasSnapshotSchema Zod at the boundary + `hasForbiddenKeyDeep` rejects __proto__/constructor/prototype in edge sourcePath/targetKey and shared_state at any depth (FOUND-6) |
| T-23-02 | Tampering | node.data spec-content injection | mitigate | Zod `.refine()` rejects `spec`/`root` keys in node.data — layout rows carry only provenance refs (D-05); specs rehydrate from chat_messages |
| T-23-03 | Information disclosure | cross-tenant chat_canvas_layouts read | mitigate | RLS deny-all RESTRICTIVE (anon + authenticated); backend service-role bypasses by design (mirrors 0023) |
| T-23-04 | Denial of service | unbounded snapshot payload | mitigate | MAX_CANVAS_NODES=200 / MAX_CANVAS_EDGES=400 caps + serialized-size guard on shared_state |
| T-23-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan (Drizzle/Zod already deps); zustand install is gated in plan 23-05 |
</threat_model>
