## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline node padding, edge label offsets |
| sm | 8px | Node internal padding, filter checkbox gaps |
| md | 16px | Filter rail section spacing, detail pane content padding |
| lg | 24px | Detail pane header padding, toolbar height (h-6 = 24px) |
| xl | 32px | Layout panel gutter |
| 2xl | 48px | Graph canvas minimum breathing room from edges |
| 3xl | 64px | Page-level side padding (when sidebar is collapsed) |

Exceptions:
- Toolbar height: 44px (h-11) — matches touch-target minimum and existing sidebar header height
- Node sizes are fixed layout units, not spacing tokens (see Node Visual Language section)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display | 24px | 600 | 1.2 |
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.25 |
| Caption | 12px | 400 | 1.4 |

Exactly 4 sizes (12 / 14 / 16 / 24) — matching Phase 9/10. No 20px tier. Section and detail-pane headings use 16px Body (`text-base`) at weight 600; true page/section titles use 24px Display (`text-2xl`) at weight 600.

Two weights only: 400 (normal) and 600 (semibold). No font-medium (500) anywhere in this phase.

Node text uses:
- Entity type name: 14px / 600 (`text-sm font-semibold`)
- Field name: 12px / 400 (`text-xs`)
- Knowledge node title: 14px / 600 (`text-sm font-semibold`)
- Node sub-label / type badge: 12px / 400 (`text-xs text-muted-foreground`)

## Page Layout Contract

### Shell
Three-zone horizontal layout inside the existing `AppSidebar` shell:

```
[Filter Rail 240px] | [Graph Canvas flex-1] | [Detail Pane 320px]
```

Top: Full-width `h-11` toolbar above the three-zone split.

Responsive: Filter rail collapses to a Sheet on mobile (not in scope for Phase 11 — mark as `// TODO: mobile`).

### Toolbar (`h-11`, `bg-background/70 backdrop-blur-md border-b border-border/50`)
- Left: page title "Knowledge" — `text-sm font-semibold`
- Right: zoom-to-fit button (Maximize2 icon), layout-toggle button (currently dagre only — button disabled if only one layout), node-count summary (`text-xs text-muted-foreground` — "{N} nodes")
- All buttons: `size="sm" variant="ghost"`, `size-4` icons

### Filter Rail (`w-60`, `bg-background/70 backdrop-blur-md border-r border-border/50`)
- Header: "Show" — `text-xs font-semibold text-muted-foreground uppercase tracking-wide px-4 pt-4 pb-2`
- Node type checkboxes (6 items): each is `flex items-center gap-2 px-4 py-1.5 text-sm`
  - Checkbox color dot (8×8px `rounded-full`) matches node role-color
  - Labels: "Entity Types", "Fields", "Instances", "Emails", "Knowledge Rules"
  - Default checked: Entity Types + Fields. Instances, Emails, Knowledge Rules unchecked by default.
- Separator `<Separator />`
- "Show instances" toggle: `Switch` component, label "Show all instances", sub-label `text-xs text-muted-foreground` "May slow rendering with large datasets"
- Bottom of rail: node count summary per type, `text-xs text-muted-foreground`, e.g. "8 types · 35 fields · 0 instances"

### Graph Canvas (`flex-1`, `bg-muted/40`)
- React Flow viewport fills the entire panel
- Controls: React Flow built-in `<Controls />` component, bottom-left, styled with `className="border border-border/50 bg-background/70 backdrop-blur-md rounded-md"`
- MiniMap: React Flow `<MiniMap />`, bottom-right, `nodeColor` mapped to role-color palette, `className="border border-border/50 bg-background/70 backdrop-blur-md rounded-md"`
- Background: React Flow `<Background />` with `variant="dots"` and `color="hsl(var(--border))"`, gap 20, size 1

### Detail Pane (`w-80`, `bg-background/70 backdrop-blur-md border-l border-border/50`)
- Default state (no node selected): centered empty state — `Share2` icon 24px muted, then "Click any node to explore it" `text-sm text-muted-foreground text-center`
- Selected node: scrollable content area (`ScrollArea` full height)
  - Header: node type badge + node label (`text-base font-semibold`)
  - Sections vary by node type (see Node Detail section below)
  - Close button: `X` icon, top-right, `variant="ghost" size="icon" size-4`

### ResizablePanelGroup
Implement with `@nauta/ui/resizable` (`ResizablePanelGroup direction="horizontal"`):
- Filter rail: `defaultSize={18}` (240px / 1280px viewport ≈ 18%)
- Graph canvas: `defaultSize={57}`
- Detail pane: `defaultSize={25}`
- Drag handles: `<ResizableHandle withHandle />`

All edges use React Flow `<BaseEdge />` with custom `EdgeLabelRenderer` for type labels.
Edge labels: `text-xs text-muted-foreground bg-background/80 px-1 rounded` (shown on hover only, not by default).
No arrows on taxonomy edges; `MarkerType.ArrowClosed` size 6 on instance/email edges.

| Edge type | Stroke | Width | Style | Animated |
|-----------|--------|-------|-------|----------|
| entity_type → entity_type_field | `hsl(164 39% 22% / 0.5)` | 1.5px | solid | false |
| entity_instance → entity_type | `hsl(262 83% 57% / 0.5)` (violet) | 1.5px | solid | false |
| email_component → entity_instance (match) | `hsl(38 92% 50% / 0.6)` (amber) | 1.5px | dashed | true |

| knowledge_node → entity_type (scope) | `hsl(164 39% 22% / 0.4)` | 1px | dashed | false |

Edge animation uses React Flow default `animated={true}` (SVG dash-offset animation). Animated edges: match + knowledge links only.

## Layout Algorithm

**Algorithm: dagre (hierarchical top-to-bottom)**
Package: `@dagrejs/dagre` (peer dep of `@xyflow/react`)

Dagre configuration:
```
rankdir: "TB"          // top-to-bottom
ranksep: 64            // 64px vertical gap between ranks
nodesep: 32            // 32px horizontal gap between nodes
edgesep: 16
```

Rank assignments:
- Rank 0: entity_type nodes (the taxonomy crown)
- Rank 1: entity_type_field nodes (radiating below each entity_type)
- Rank 2: entity_instance nodes (when visible)
- Rank 3: email_component nodes (when visible)
- Rank 4: email nodes (when visible)
- knowledge_nodes: pinned at rank 0–1, placed to the right of the rightmost entity_type

Recompute layout on: filter change, "Show instances" toggle, viewport mount.
Animate transition with React Flow `fitView` + 300ms ease-in-out after layout recompute.

## Default Render Depth (Critical Empty-State Contract)

**Initial state when page loads (staging: 0 knowledge_nodes, 0 entity_instances today):**

Always render: entity_type nodes + entity_type_field nodes (the taxonomy).
This produces the "what we extract" map — 8 entity types + ~35 fields in a legible hierarchy.
This is NOT an empty state. It is the primary value-delivery of this phase.

Below the graph canvas, render the taxonomy explainer banner:
- Position: `absolute bottom-0 left-0 right-0` inside the graph canvas panel (above MiniMap)
- Style: `bg-background/70 backdrop-blur-md border-t border-border/50 px-4 py-2`
- Copy: "Your extraction schema — {N} entity types, {M} fields. Instances and knowledge rules appear as emails are processed."
- `text-xs text-muted-foreground`
- Dismiss button: `X` icon, `variant="ghost" size="icon"` size-3. Dismissed state persists in `localStorage` key `nauta.knowledge.taxonomy-banner-dismissed`.

**Instances hidden by default.** Shown when user enables "Show instances" toggle in filter rail.
Threshold: if total entity_instance + email_component + email count < 50, auto-show without toggle.

**knowledge_nodes hidden when count = 0.** When knowledge_nodes.length > 0, they appear automatically.

## States and Interactions

### Loading state
Show 8 ghost nodes in a soft radial cluster:
- 3 tall ghost rectangles (entity_type shape) at top — `<Skeleton className="h-12 w-40 rounded-lg" />`
- 5 short ghost rectangles (entity_type_field shape) below — `<Skeleton className="h-8 w-32 rounded-md" />`
- No edges during skeleton
- Skeleton nodes positioned via CSS `translate` into a rough 3-up / 5-below arrangement (not React Flow — just a static div)

### Error state
Full-canvas centered error:
- Icon: `AlertCircle` (lucide), `size-8 text-destructive`
- Heading: "Could not load the knowledge graph." — `text-base font-semibold`
- Body: "Try refreshing the page." — `text-sm text-muted-foreground`
- Button: "Refresh page" — `variant="outline" size="sm"` — calls `window.location.reload()`

### No schema state (entity_types.length === 0)
Replaces the graph canvas entirely:
- Icon: `Shapes` (lucide), `size-8 text-muted-foreground`
- Heading: "No schema defined yet." — `text-base font-semibold`
- Body: "Add entity types to see your knowledge network." — `text-sm text-muted-foreground`
- No button (entity type creation is out of scope for this page, D-09)

### Node click
- Clicking a node: sets selected node in local React state, populates detail pane, applies `ring-2 ring-primary ring-offset-1` to the node
- Clicking canvas background: deselects node, returns detail pane to default "Click any node" state
- Clicking a node already selected: no change (does not deselect)

### Zoom + pan
- Standard React Flow behavior (scroll to zoom, drag to pan)
- "Fit view" button in toolbar: calls `reactFlowInstance.fitView({ padding: 0.1, duration: 300 })`
- Double-click on node: calls `reactFlowInstance.fitView({ nodes: [node], padding: 0.5, duration: 300 })`

## Accessibility Contracts

Mirrors the Phase 9/10 accessibility tables — same token system, no new tokens.

### Icon-only controls — required `aria-label`

| Element | ARIA role | Label |
|---------|-----------|-------|
| Toolbar zoom-to-fit button (`Maximize2`) | `button` | `aria-label="Zoom to fit"` |
| Toolbar layout-toggle button | `button aria-pressed={isDagre}` | `aria-label="Toggle layout"` |
| Detail pane close button (`X`) | `button` | `aria-label="Close detail panel"` |
| Taxonomy banner dismiss button (`X`) | `button` | `aria-label="Dismiss"` |
| React Flow built-in Controls | (library-provided) | inherit React Flow `aria-label`s; do not strip |

### Graph canvas + nodes

| Element | ARIA role | Label / behavior |
|---------|-----------|------------------|
| Graph viewport | `role="application" aria-label="Knowledge graph"` | — |
| Node (all 6 types) | `role="button" tabIndex={0} aria-pressed={isSelected}` | accessible name = node label text (entity type name, field name, instance name, component type, email subject, knowledge rule title) prefixed by its type, e.g. `aria-label="Entity Type: Invoice"` |
| Node keyboard activation | — | `Enter` / `Space` selects the focused node (same effect as click); selection updates detail pane |
| Edge | `role="img" aria-label="{source} relates to {target}"` | non-interactive; descriptive only |
| Detail pane container | `role="complementary" aria-label="Node details"` | — |
| Detail pane on selection | `aria-live="polite"` on the content region | announces the newly selected node's type + name when selection changes |
| Taxonomy explainer banner | `role="status" aria-live="polite"` | — |
| Loading skeleton | `role="status" aria-busy="true" aria-label="Loading knowledge graph"` | — |
| Error state | `role="alert"` | error heading + body announced |

### Focus and keyboard traversal

- Keyboard focus order: TOOLBAR → FILTER RAIL → GRAPH CANVAS (tab stop enters the node group) → DETAIL PANE.
- Within the GRAPH CANVAS, `Tab` / `Shift+Tab` moves between focusable nodes in dagre rank order (rank 0 left-to-right, then rank 1, …). React Flow nodes are keyboard-focusable (`tabIndex={0}`) and carry accessible names per the node table above.
- Focus-visible ring on interactive nodes reuses the existing selection treatment: `focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-1` (same `ring-2 ring-primary` token as the selected state — no new tokens).
- Filter rail checkboxes, the "Show instances" `Switch`, and detail-pane links are reachable in DOM order via standard Radix/shadcn keyboard support.
- `Escape` while a node is selected deselects it and returns the detail pane to the default "Click any node to explore it" state; focus returns to the previously focused node.

## Implementation Notes for Executor

1. **Client island**: wrap the entire graph canvas in `dynamic(() => import('./_components/knowledge-graph'), { ssr: false, loading: () => <KnowledgeGraphSkeleton /> })` in the page component. Pattern: same as Phase 5 react-pdf island.

2. **tRPC router**: create `packages/api-client/src/router/knowledge/` with `graph.ts` (returns nodes + edges), `list.ts`, `detail.ts`, and `index.ts`. Pattern mirrors `entities/` split exactly.

3. **knowledge_node_edges table**: migration creates the table but Phase 11 writes 0 rows. Edge data for this phase is derived from existing FK relationships (entity_type_fields.entity_type_id, entity_instances.entity_type_id, email_components.entity_instance_id, email_components.email_id). The provider boundary (D-11) means the graph data API returns a unified `{ nodes, edges }` shape regardless of whether edges come from FKs or the knowledge_node_edges table.

4. **Dagre import**: `import dagre from "@dagrejs/dagre"` — confirm this package is already a dep of `@xyflow/react` in the project's lockfile before adding explicitly.

5. **No `font-medium` (500)**: use only `font-normal` (400) or `font-semibold` (600). This matches Phase 9/10 constraint.

6. **Tenant isolation**: `importer_id` filter applied server-side in tRPC router from session context, never from URL param.
