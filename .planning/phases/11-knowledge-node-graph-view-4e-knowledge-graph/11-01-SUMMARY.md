## What Was Built

### Task 1: Schema + Migration (BLOCKING gate)

`packages/db/src/schema/knowledge-node-edges.ts` defines the `knowledge_node_edges` table with 8 columns:
- `id` (uuid PK defaultRandom)
- `source_node_id` (uuid FK → knowledge_nodes.id ON DELETE CASCADE)
- `target_ref_id` (uuid, no `.references()` — polymorphic)
- `target_ref_type` (text nullable)
- `relation_type` (text notNull default 'related')
- `confidence` (real notNull default 1.0)
- `source` (text notNull default 'manual')
- `created_at` (timestamp with timezone notNull defaultNow)

Two indexes: `idx_knowledge_node_edges_source_node_id`, `idx_knowledge_node_edges_target_ref_id`.

Migration `0019_cold_energizer.sql` generated by drizzle-kit, applied locally, and asserted via `assert-knowledge-node-edges.ts` querying `information_schema.tables`.

**PENDING DEPLOY:** `npm run migrate:staging` and `npm run migrate:prod` must be run before the next web/listener deploy to staging and production Supabase instances (refs: fyfwkjvbcrmjqjysdyqw, dazyccjijdahxyciptkp).

### Task 2: knowledge tRPC Router

Four files under `packages/api-client/src/router/knowledge/`:

**graph.ts** — `knowledge.graph` procedure:
- Always returns entity_type + entity_type_field nodes + field→type edges (D-04)
- `includeInstances=true`: adds entity_instance nodes + component↔instance edges via `ComponentEntityCandidateLinks` JOIN (schema discrepancy: no direct FK on email_components)
- `includeEmails=true`: adds email_component + email nodes + nesting edges
- Always appends knowledge_node nodes (when count > 0) + component↔knowledge_node edges
- **D-11 seam:** reads `KnowledgeNodeEdges`, maps rows to `GraphEdge` shape, unions into edge list (0 rows today, seam is live)
- Exports `graphInputSchema`, `shapeGraphResponse`, `GraphNode`, `GraphEdge`, `GraphResponse` for DB-free testing

**list.ts** — `knowledge.list` procedure with limit+1 pagination, offset, optional importerId filter.

**detail.ts** — `knowledge.byId` procedure returning node + outgoing edges or null.

**index.ts** — `knowledgeRouter` composing all three procedure sets.

`root.ts` updated to register `knowledge: knowledgeRouter`.

**22 DB-free tests** (102 total passing):
- `graph.test.ts`: 15 tests — shapeGraphResponse immutability/identity, graphInputSchema enum/defaults/bounds (6 node-type allow-list)
- `list.test.ts`: 7 tests — listKnowledgeInputSchema limit/offset/importerId bounds
