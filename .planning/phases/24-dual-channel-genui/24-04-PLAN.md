<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model-authored fields/submitLabel → form engine | untrusted declaration drives form rendering |
| user-typed field values → agent context | untrusted values flow to the model after validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-20 | Tampering | model declares a permissive response schema | mitigate | The response schema is DERIVED server-side from the fields (Task 1) — the model never authors it; additionalProperties:false always |
| T-24-21 | Injection | typed field values reach model context | mitigate | Values validated against the derived schema (24-02 D-10) and attributed as the delimited interaction_result user turn — plain data, never tool-block smuggling |
| T-24-22 | Tampering | values payload enrichment breaks existing handlers | mitigate | Built-in setState handler + panel-action-bridge narrowing ignore unknown payload keys (regression-asserted in Task 2) |
| T-24-23 | Spoofing | client submits values for non-declared fields | mitigate | additionalProperties:false in the derived schema → 422; the form engine only renders declared fields anyway |
| T-24-24 | Denial of Service | unbounded fields array | mitigate | input_schema caps fields at 12 items; oversized tool JSON already bounded by max_tokens |
| T-24-SC | Supply chain | new packages | mitigate | none introduced this plan |
</threat_model>
