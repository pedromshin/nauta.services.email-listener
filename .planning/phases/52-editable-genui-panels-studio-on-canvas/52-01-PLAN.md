<objective>
Build the shared, no-migration substrate every editable-panel feature depends on: (1) the
per-panel **overlay data model** (pack override + supersede-never-mutate version chain) with
pure resolution/mutation helpers, (2) an app-owned **theming wrapper** that applies a pack +
bounded token overrides as inline CSS variables, and (3) the **read/write/persist plumbing**
that stores overlays under the existing `chat_canvas_layouts.sharedState` column so edits
rehydrate on reload — **without any DB migration** (Docker/WSL is down this session).

Purpose: PANL-01..04 all read the active spec/pack from an overlay and write new versions back.
Defining these contracts first (interface-first) lets the four feature plans implement against a
stable API instead of re-deriving canvas internals.
Output: `panel-overlay.ts`, `panel-theme-scope.tsx`, `panel-overlay-context.tsx` + tests, and the
`CanvasPersistenceProvider` wiring in `chat-canvas.tsx`.
</objective>

Canvas store (apps/web/src/app/chat/_canvas/canvas-store.ts) — VERBATIM:
- `values: Record<string, unknown>` is `{ panels: {...}, shared: {...} }`.
- `mutate(mutation, path, value)` applies only the bounded enum
  `["toggle","set","reset","increment","decrement"]`; dotted paths crossing
  `__proto__/constructor/prototype` are no-ops. Use `mutate("set", "shared.panelOverlays."+panelId, next)`.
- `read(path)` / `resolveCanvasPath(values, path)` read a dotted path (bail-to-undefined).
- Seed rehydration (canvas-store-context.tsx `toCanvasStoreSeed`) restores ONLY the
  `panels` and `shared` keys from persisted sharedState — so overlays MUST live UNDER `shared`
  (i.e. `shared.panelOverlays.{panelId}`) to survive reload. A new top-level key would persist
  but NOT rehydrate.

Persistence (apps/web/src/app/chat/_canvas/use-canvas-persistence.ts + chat/canvas.ts):
- `scheduleSave(canvasStore?)` arms a debounced (~800ms) `chat.saveCanvasLayout` that snapshots
  `canvasStore.getState().values` into `chat_canvas_layouts.sharedState`.
- Server gate (canvas-schema.ts `sharedStateSchema`): permissive `z.record(z.string(), z.unknown())`
  + deep `__proto__/constructor/prototype` guard + `MAX_SHARED_STATE_SERIALIZED_CHARS = 100_000`.
  Store spec content as a JSON **string** (not a parsed object) so the deep guard never traverses it
  and the size guard stays predictable.

Spec envelope (packages/genui/src/schema/spec-schema.ts):
- `SpecRootSchema` (.strict) carries optional `style_pack_id: StylePackIdSchema`. Use
  `SpecRootSchema.safeParse(JSON.parse(specJson))` to read a spec's own pack.

Theme (packages/genui/src/theme): `getStylePack(id)` returns `{ resolvedVars: Record<string,string> }`
(cssVarName-without-`--` → value, HSL channel strings / raw radius/shadow/font values);
`DEFAULT_PACK_ID = "polytoken-teal"`; `STYLE_PACK_IDS`; type `StylePackId`. `ThemedRoot`
(themed-wrapper.tsx) is the precedent — PanelThemeScope mirrors it but is app-owned AND accepts
optional overrides.

Provenance (apps/web/src/app/chat/_canvas/node-data-schemas.ts): `ProvenanceSchema`
`{ messageId: uuid, partIndex: int>=0, runId: uuid|null }`.

chat-canvas.tsx already computes `canvasStore` and `persistence.scheduleSave` and holds
`conversationId`; it wraps the tree with `CanvasStoreProvider`/`CanvasEdgesProvider`/`CanvasSpecProvider`.

    - resolveActivePanel(overlay|undefined, baseSpecJson, baseIsStreaming): when baseIsStreaming
      OR overlay is undefined/has no active version → returns { specJson: baseSpecJson, packId:
      (overlay.stylePackId ?? base spec's style_pack_id ?? DEFAULT_PACK_ID), tokenOverrides: {} }.
      When an active version exists and not streaming → { specJson: version.specJson, packId:
      (overlay.stylePackId ?? version.stylePackId ?? DEFAULT_PACK_ID), tokenOverrides: version.tokenOverrides ?? {} }.
    - setPack(overlay|undefined, packId): returns overlay with stylePackId=packId (creates a minimal
      overlay if undefined). Does NOT add a version.
    - appendVersion(overlay|undefined, input): appends input as a new version (fresh uuid,
      createdAt ISO, parentVersionId = prior activeVersionId), sets activeVersionId to it, and CLEARS
      stylePackId to null (the new version's own pack takes over). Immutable — never mutates input overlay.
    - restoreVersion(overlay, versionId): supersede-never-mutate — appends a NEW version cloning the
      target version's specJson/stylePackId/tokenOverrides (generatedBy copied from target,
      parentVersionId = target.id), sets it active, clears stylePackId. No-op returning overlay unchanged
      if versionId is unknown.
    - listPriorVersions(overlay): every version EXCEPT the current active one, newest-first, for the
      history UI. Empty array when overlay undefined or only the active version exists.
    - parseOverlay(raw): PanelOverlaySchema.safeParse → data | undefined (degrade-not-throw, T-23-09 posture).
  </behavior>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → canvas store → sharedState (persisted) | overlay JSON the client writes is persisted and re-read on reload |
| persisted sharedState → client (rehydrate) | a tampered/legacy row must degrade, never crash |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-01-01 | Tampering | overlay written to `shared.panelOverlays` | mitigate | server `sharedStateSchema` already deep-guards `__proto__/constructor/prototype` + caps 100k chars; `parseOverlay` safeParses on read and degrades malformed data to `undefined` (never trusts a bad row) |
| T-52-01-02 | Denial of Service | unbounded version growth in sharedState | mitigate | `versions` capped at 50, `specJson` capped 60k; over-budget save fails gracefully via the existing `buildSnapshot` try/catch → `saveStatus="error"` (never crashes the canvas). Documented budget bound; dedicated versions table deferred (no migration tonight) |
| T-52-01-03 | Tampering | spec content stored in sharedState (D-05 tension) | accept | D-05 guards `node.data`, not `sharedState`; overlays store FINALIZED (non-streaming) spec strings deliberately, so the streaming-relayout concern D-05 addresses does not apply. Strings are never deep-traversed |
| T-52-SC | Tampering | npm installs | mitigate | ZERO new dependencies this plan (all primitives/exports already vendored) — no install task, nothing to vet |
