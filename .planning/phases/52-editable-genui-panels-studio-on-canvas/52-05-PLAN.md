<objective>
Deliver the **server side of PANL-04**: a one-shot natural-language → `{style_pack_id, token_overrides}`
resolution using the **existing Bedrock transport** (reused via a new `AsyncAnthropicBedrock`
forced-tool-use adapter, mirroring the genui generator), exposed as `POST /v1/genui/retheme` and

the resolution; Plan 52-06 wires the client popover + applies the result as a `retheme` version.

role. Verification is pytest with a **mocked resolver** + unit-tested prompt assembly, and vitest with
**mocked fetch** for the tRPC boundary. One real Bedrock smoke call is acceptable but not required; the
real end-to-end (FastAPI up) is queued to MORNING-CHECKLIST §G.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user instruction → Bedrock | free-text NL instruction sent to the model |
| Bedrock/LLM output → FastAPI → tRPC → client | model output is untrusted and must be schema-gated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-05-01 | Tampering | LLM-chosen style_pack_id | mitigate | Python constrains via forced-tool input_schema + is_known_pack_id; Node `RethemeResolutionSchema` re-validates against `z.enum(STYLE_PACK_IDS)` — the authoritative gate (GEN-03/D-08) |
| T-52-05-02 | Tampering | LLM token_overrides (arbitrary CSS injection) | mitigate | keys refined to ALLOWED_OVERRIDE_KEYS only; color values matched against a strict HSL-channel regex, radius/density against raw-value guards; anything else rejected at the web boundary. PanelThemeScope only ever sets `--{key}` inline (no raw HTML) |
| T-52-05-03 | Information Disclosure | raw FastAPI/Bedrock error leaking | mitigate | non-2xx/exception → friendly `reason`; raw detail to server-side `logError` only (generate.ts pattern) |
| T-52-05-04 | Denial of Service | oversized / looping generation | mitigate | one-shot (no repair loop, locked); instruction capped 280 chars at the tRPC boundary |
| T-52-SC | Tampering | package installs | mitigate | ZERO new npm deps; Python reuses the already-vendored `anthropic`/`AsyncAnthropicBedrock` client (no new Python dep) — nothing to vet |
</threat_model>
