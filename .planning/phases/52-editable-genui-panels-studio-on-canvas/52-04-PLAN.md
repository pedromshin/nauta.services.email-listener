<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FastAPI generate output → panel version | a regenerated spec must be validated before it becomes a version |
| overlay version content → restore | restoring must not resurrect an invalid spec |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-04-01 | Tampering | regenerated spec from FastAPI | mitigate | `genui.generate` already re-validates via `SpecRootSchema.safeParse` at the web boundary (generate.ts) before the control ever sees it; only `outcome!=="fallback"` results are stored |
| T-52-04-02 | Tampering | restored version spec | mitigate | versions were validated when appended; the render path (GenuiPartBoundary) re-runs SpecRootSchema.safeParse anyway (FOUND-6) so a corrupted stored spec degrades to SAFE_FALLBACK, never crashes |
| T-52-04-03 | Repudiation | provenance integrity | mitigate | every version records generatedBy + parentVersionId + createdAt; restore appends (never rewrites) so history is append-only |
| T-52-SC | Tampering | npm installs | mitigate | ZERO new dependencies — no install, nothing to vet |
</threat_model>
