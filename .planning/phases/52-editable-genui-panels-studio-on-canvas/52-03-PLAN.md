<objective>
Deliver **PANL-02**: in-place, bounded, schema-validated editing of a genui panel's spec parameters
through the Parameter Editor Popover (52-UI-SPEC Component 2), gated server-side by the **same
untrusted-input boundary as FOUND-6** (`SpecRootSchema.safeParse` + whitelist-only application). No
free-form JSON is ever offered; an invalid save shows an inline banner with values preserved (no
partial apply). A successful save appends an `edit` version to the panel overlay.

Purpose: turn read-only panels into editable surfaces without opening a spec-injection hole — the
client sends only bounded, Zod-validated param values; the server re-validates the whole spec.
Output: the authoritative param schema + pure application function, the `genui.applyPanelEdit`
procedure, and the popover control (replacing Plan 02's skeleton).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client form → applyPanelEdit input | user-typed param values cross into spec construction |
| client-supplied currentSpecJson → server | a spec string the client sends must not be trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-03-01 | Tampering | param values | mitigate | `PanelEditParamsSchema` (.strict, bounded, enum) at the tRPC boundary; only whitelisted keys valid for the root type are applied; no free-form JSON control offered (UI-SPEC locked) |
| T-52-03-02 | Elevation of Privilege | client-supplied spec smuggling arbitrary nodes | mitigate | server re-parses currentSpecJson via `SpecRootSchema.safeParse` (FOUND-6) AND re-validates the post-apply result; `applyWhitelistedParams` only ever patches root attrs, never children/structure |
| T-52-03-03 | Tampering | prototype pollution via params | mitigate | `.strict()` schemas reject `__proto__/constructor/prototype` keys; enum/number fields cannot carry object payloads |
| T-52-03-04 | Information Disclosure | raw validation errors leaking to client | mitigate | procedure returns friendly `reason` only; raw JSON.parse/Zod errors go to server-side `logError` (generate.ts pattern) |
| T-52-SC | Tampering | npm installs | mitigate | ZERO new dependencies — no install, nothing to vet |
</threat_model>
