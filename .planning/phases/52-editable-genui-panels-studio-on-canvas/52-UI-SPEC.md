# Phase 52 — UI Design Contract: Editable Genui Panels / Studio-on-Canvas

> Visual and interaction contract for turning a canvas genui panel into a live editing
> surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker. **This phase is a
> CHROME EXTENSION, not a redesign** — it consumes the Phase-48 token system and the
> Phase-51 interactive-state convention unchanged, and extends the existing
> `GenuiPanelNode` (`apps/web/src/app/chat/_canvas/genui-panel-node.tsx`, locked by
> 23-UI-SPEC/26-UI-SPEC FIX-04) with a new toolbar row + three popover surfaces. No new
> npm dependency, no new vendored `@polytoken/ui` component — every primitive used below
> (`Select`, `Popover`, `Tooltip`, `Button`, `Input`, `Textarea`, `Switch`, `Label`,
> `ScrollArea`, `Badge`) already exists in the package.

Unattended overnight run (52-CONTEXT.md `Mode`): every open design question below is
resolved with an explicit, documented judgment call (see **Judgment Calls** section) rather
than left ambiguous for the executor.

## Spacing Scale

Inherited unchanged from Phase 26/28/51 (not reopened this phase):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), popover field spacing |
| sm | 8px | Row padding (`px-2`, `py-2`), popover-footer gaps |
| md | 16px | Popover section padding (`p-4` default `PopoverContent`) |
| lg | 24px | (not used by this phase's new chrome — canvas-level only) |

Exceptions specific to this phase:

- **New `.panel-toolbar` row is `h-8` (32px)**, and its icon buttons are `size-6` (24px) —
  below the 44px touch-target floor guard. This is consistent with the existing carve-out
  already recorded in 51-UI-SPEC.md's Spacing Scale ("this phase does not need to apply
  \[the 44px floor\] proactively... any interactive element re-skinned this phase must not
  end up computing below 44px if it was already compliant") — `GenuiPanelNode`'s existing
  header-row controls (e.g. `KnowledgePreviewNode`'s remove button) are already `size-6`,
  so this phase matches the established desktop-canvas density rather than inventing a
  denser or looser one. Phase 53 (mobile) owns making canvas chrome touch-safe.
- **`GenuiPanelNode`'s outer shell `min-h-[240px]` becomes `min-h-[272px]`** (+32px) to
  make room for the new toolbar row without shrinking the rendered spec's original 204px
  content viewport. `min-w-[320px]` is unchanged. See Judgment Calls #2.

## Component Inventory — New Chrome

### 1. Panel Toolbar Row (hosts PANL-01/02/03/04's entry points)

A new **second header row**, inserted directly below the existing `.node-drag-handle` h-9
row and above the `ScrollArea` body, in `GenuiPanelNode`'s `GenuiPanelNodeBody`. It is
**not** part of the drag handle (26-UI-SPEC FIX-04's "the h-9 row is the ONLY drag handle"
stays untouched — see Judgment Calls #1).

```
<div role="toolbar" aria-label="Panel actions"
     className="flex h-8 shrink-0 items-center justify-between gap-1 border-b border-border/60 bg-background px-2">
  {/* left: pack switcher */}
  <Select value={stylePackId} onValueChange={...} disabled={isMutating}>
    <SelectTrigger aria-label="Style pack"
      className="h-6 w-28 shrink-0 gap-1 rounded-md border-none bg-transparent px-1.5 text-xs font-normal text-muted-foreground shadow-none hover:bg-accent hover:text-accent-foreground focus:ring-0 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-1 disabled:cursor-not-allowed disabled:opacity-40">
      <SelectValue />
    </SelectTrigger>
    <SelectContent>
      {STYLE_PACK_IDS.map((id) => (
        <SelectItem key={id} value={id}>{STYLE_PACKS[id].label}</SelectItem>
      ))}
    </SelectContent>
  </Select>

  {/* right: action cluster */}
  <div className="flex items-center gap-1">
    <IconButton icon={SlidersHorizontal} label="Edit parameters" ... />
    <IconButton icon={RotateCw}          label="Regenerate" ... />
    <IconButton icon={Wand2}             label="Re-theme" ... />
    <IconButton icon={History}           label="Version history" ... />
  </div>
</div>
```

Shared icon-button class (all 4 action buttons — mirrors `KnowledgePreviewNode`'s existing
remove-button recipe, neutral hover instead of destructive hover):

```
"flex size-6 shrink-0 items-center justify-center rounded-md text-muted-foreground transition-colors hover:bg-accent hover:text-accent-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-1 disabled:opacity-40 disabled:pointer-events-none"
```

Icon size `size-3.5` inside each `size-6` button (matches `KnowledgePreviewNode`'s `X`
icon sizing). Each button is wrapped in `Tooltip`/`TooltipTrigger`/`TooltipContent` (one
shared `TooltipProvider` around the whole action cluster, `delayDuration={300}`), mirroring
`add-knowledge-preview-popover.tsx`'s exact Tooltip+Popover composition.

**Pack-switcher persistence (PANL-01):** `onValueChange` optimistically sets the local
value, fires the mutation, and persists silently on success (per 52-CONTEXT.md "switching
applies immediately and persists" — no confirmation step). On mutation failure, revert the
`Select` to the prior value and show `toast.error` (see Interactive-State Contract).

**Mutual exclusion:** while any one of the 4 actions (or the pack switch) is pending for
this panel, the `Select` and all 4 buttons are `disabled` **except** the one actually in
flight, which shows its own busy affordance instead of the dimmed-disabled state (see
Interactive-State Contract table).

### 2. Parameter Editor Popover (PANL-02)

Trigger: the `SlidersHorizontal` toolbar button (`PopoverTrigger asChild`). Anchored
`align="start" side="bottom"`.

```
<PopoverContent className="w-80 space-y-3">
  <p className="text-xs font-semibold text-foreground">Edit panel parameters</p>
  {serverError && (
    <div role="alert" className="rounded-md border border-destructive/30 bg-destructive/5 p-2 text-xs text-destructive">
      Couldn't save these changes — check the highlighted fields.
    </div>
  )}
  <div className="space-y-3">
    {/* one row per whitelisted spec parameter — field-type mapping below */}
  </div>
  <div className="flex justify-end gap-2 pt-1">
    <Button type="button" variant="ghost" size="sm" onClick={handleCancel}>Cancel</Button>
    <Button type="button" variant="default" size="sm" disabled={!isDirty || !isValid || isPending} onClick={handleSave}>
      {isPending ? <><Loader2 className="mr-1 size-3.5 animate-spin" aria-hidden />Saving…</> : "Save changes"}
    </Button>
  </div>
</PopoverContent>
```

**Field-type → control mapping** (the exact spec-parameter whitelist is Claude's
Discretion at plan time per 52-CONTEXT.md; this table is the binding VISUAL contract for
whatever fields the planner selects):

| Zod field shape | Control | Row pattern |
|---|---|---|
| Bounded string (`z.string().max(n)`) | `Input` (text) | `Label` above, `Input` below, `"{n} characters max"` helper text if `maxLength` is finite |
| Bounded long-form string | `Textarea` (`rows={3}`) | Same as above |
| Enum / string union (`z.enum([...])`) | `Select` (default `h-9` sizing — full-width form control, **not** the compact toolbar recipe above) | `Label` above, `Select` below |
| Bounded number (`z.number().min().max()`) | `Input type="number"` with `min`/`max`/`step` HTML attrs | `Label` above, `Input` below, `"Range: {min}–{max}"` helper text |
| Boolean (`z.boolean()`) | `Switch` | `Label` left, `Switch` right, single row (`flex items-center justify-between`) |

No free-form JSON textarea is ever offered (52-CONTEXT.md, locked decision). No `Slider`
control — not vendored in `@polytoken/ui` yet; see Judgment Calls #4.

Client-side Zod validation runs on blur/change; a failing field shows
`<p className="text-xs text-destructive">{message}</p>` directly below that field's
helper text. The server-side FOUND-6 gate is the second, authoritative check — its
rejection surfaces as the top-of-form `serverError` banner shown above, and the form
**never** clears on a server rejection (values stay exactly as typed so the user can fix
and resubmit — no partial apply).

### 3. Regenerate (PANL-03, one-click — no popover)

Clicking the `RotateCw` toolbar button immediately triggers a new-version regeneration of
the panel's current variant. No confirmation, no form. While pending: icon gets
`motion-safe:animate-spin`, button `aria-label` swaps to `"Regenerating panel"`, button
`disabled`, and the whole `GenuiPanelNode` shell is wrapped in `<GeneratingRing active
className="rounded-lg">` (see Judgment Calls #5) for the duration. On success, the panel's
rendered content swaps to the new variant in place — no toast, no extra chrome (the
visible content change is itself the confirmation, consistent with `SaveStatusIndicator`'s
"ambient, never blocking" register).

### 4. Version History Popover (PANL-03)

Trigger: the `History` toolbar button. Anchored `align="end" side="bottom"`. Minimal list,
no diff view (52-CONTEXT.md, locked).

```
<PopoverContent className="w-80 p-0" align="end">
  <div className="border-b border-border/60 px-3 py-2">
    <p className="text-xs font-semibold text-foreground">Version history</p>
  </div>
  <ScrollArea className="max-h-64">
    <ul role="list" aria-label="Panel versions">
      {/* topmost row: current version, always present */}
      <li className="flex items-center justify-between gap-2 border-b border-border/50 px-3 py-2 text-xs">
        <span className="flex min-w-0 items-center gap-1.5 text-muted-foreground">
          <PanelsTopLeft className="size-3 shrink-0" aria-hidden />
          Current
        </span>
        <Badge variant="outline" className="shrink-0 text-[10px] px-1 py-0">Current</Badge>
      </li>
      {/* prior versions, newest first, each with a Restore action */}
      {priorVersions.map((v) => (
        <li key={v.id} className="flex items-center justify-between gap-2 border-b border-border/50 px-3 py-2 text-xs">
          <span className="flex min-w-0 items-center gap-1.5 text-muted-foreground">
            <VersionIcon generatedBy={v.generatedBy} className="size-3 shrink-0" aria-hidden />
            <span className="truncate">{verbFor(v.generatedBy)} · {formatRelativeTime(v.createdAt)}</span>
          </span>
          <Button type="button" variant="ghost" size="sm" className="h-6 shrink-0 px-2 text-xs"
                  disabled={isRestoring} onClick={() => handleRestore(v.id)}>
            {isRestoringThisRow ? <Loader2 className="size-3.5 animate-spin" aria-hidden /> : "Restore"}
          </Button>
        </li>
      ))}
      {priorVersions.length === 0 && (
        <li className="px-3 py-4 text-center text-xs text-muted-foreground">
          No earlier versions yet — changes will appear here.
        </li>
      )}
    </ul>
  </ScrollArea>
</PopoverContent>
```

**`formatRelativeTime`** is the exact helper already defined in
`apps/web/src/app/studio/_components/history-island.tsx` ("just now" / "{n} minute(s)
ago" / "{n} hour(s) ago" / "{n} day(s) ago") — reuse it verbatim (extract to a shared
util if the planner prefers, but the strings must match exactly; do not invent a second
relative-time vocabulary).

**Provenance icon+verb mapping** (`generated_by`, from 52-CONTEXT.md's provenance fields —
reuses the SAME icon vocabulary as the toolbar buttons, so a user learns one icon set):

| `generated_by` | Icon | Verb |
|---|---|---|
| `regenerate` | `RotateCw` | "Regenerated" |
| `retheme` | `Wand2` | "Re-themed" |
| `edit` | `SlidersHorizontal` | "Edited" |
| (initial generation, if it ever appears as the oldest row) | `PanelsTopLeft` | "Generated" |

### 5. NL Re-theme Popover (PANL-04)

Trigger: the `Wand2` toolbar button. Anchored `align="end" side="bottom"`.

```
<PopoverContent className="w-72 space-y-3">
  <p className="text-xs font-semibold text-foreground">Describe a new look</p>
  {inlineError && (
    <div role="alert" className="rounded-md border border-destructive/30 bg-destructive/5 p-2 text-xs text-destructive">
      Couldn't apply that look — try describing it differently.
    </div>
  )}
  <div className="space-y-1">
    <Textarea rows={3} maxLength={280} readOnly={isPending}
      placeholder='e.g. "Make it feel more playful and colorful"'
      value={instruction} onChange={...} />
    <p className="text-right text-xs text-muted-foreground">{instruction.length}/280</p>
  </div>
  <div className="flex justify-end gap-2">
    <Button type="button" variant="ghost" size="sm" onClick={handleCancel}>Cancel</Button>
    <Button type="button" variant="default" size="sm" disabled={!instruction.trim() || isPending} onClick={handleApply}>
      {isPending ? <><Loader2 className="mr-1 size-3.5 animate-spin" aria-hidden />Re-theming…</> : <><Wand2 className="mr-1 size-3.5" aria-hidden />Apply</>}
    </Button>
  </div>
</PopoverContent>
```

280-character cap is a judgment call (Judgment Calls #6) to keep the instruction to one
concrete idea and bound generation cost — not a schema requirement from 52-CONTEXT.md.

One-shot resolution (52-CONTEXT.md, locked): no repair loop, no screenshot judging. A
schema-invalid or LLM-failure result shows the inline banner above (form stays open,
instruction text preserved) — never a partial/silent apply.

## Interactive-State Contract

Every new element follows D-48-06 (hover/active) and the focus-visible rule unchanged
(51-UI-SPEC §Interactive-State Contract — not restated here). This table is the
**feedback contract** specific to this phase's 5 mutating actions:

| Action | Success feedback | Error feedback | In-flight feedback |
|---|---|---|---|
| Switch pack (PANL-01) | `Select` shows the new value immediately (optimistic); persists silently, no extra chrome | `toast.error("Couldn't switch style — try again.", { action: { label: "Retry", onClick } })`; `Select` reverts to the prior value | `Select` `disabled`, `aria-busy="true"` |
| Save parameters (PANL-02) | Popover closes; panel content re-renders with the new params | Inline banner inside the popover (stays open, values preserved) — see Component 2 | `Save changes` button: `Loader2` spinner + "Saving…" label, `disabled` |
| Regenerate (PANL-03) | Panel content visibly swaps to the new variant; `<GeneratingRing>` deactivates — no toast | `toast.error("Couldn't regenerate this panel — try again.", { action: { label: "Retry", onClick } })` | `RotateCw` `motion-safe:animate-spin`; whole panel wrapped in `<GeneratingRing active>`; toolbar `disabled` except this button |
| Apply re-theme (PANL-04) | Popover closes; `toast.success("Panel re-themed")`; panel content re-renders on the new pack/tokens; `<GeneratingRing>` deactivates | Inline banner inside the popover (stays open, instruction preserved) — see Component 5 | `Apply` button: `Loader2` spinner + "Re-theming…" label, `disabled`; `Textarea` `readOnly`; panel wrapped in `<GeneratingRing active>` |
| Restore version (PANL-03) | History popover closes; `toast.success("Restored to an earlier version")`; panel content re-renders | `toast.error("Couldn't restore that version — try again.", { action: { label: "Retry", onClick } })` (popover stays open) | Clicked row's `Restore` button shows `Loader2`; every other `Restore` button in the list `disabled` |

`toast.error`/`toast.success` calls use the existing `sonner` import (`import { toast }
from "sonner"`) and the exact `{ action: { label, onClick } }` shape already established
in `confirm-deny-controls.tsx`'s Undo toast — no new toast library, no new toast
positioning/duration convention.

**Empty/disabled edge case:** if a panel's whitelisted parameter set is empty for its spec
type, the `SlidersHorizontal` toolbar button is `disabled` outright (not clickable into an
empty popover), with tooltip copy "This panel has no editable parameters".

## Judgment Calls (unattended-run discretion resolutions)

Explicit record per the constraints — each is a deliberate design decision, not an
oversight, made because 52-CONTEXT.md left it to "Claude's Discretion" or because
implementing PANL-01..04 required a call CONTEXT didn't make:

1. **Toolbar placement:** a NEW secondary `h-8` non-drag row below the existing `h-9`
   `.node-drag-handle` header, not packed into that header row. Keeps 26-UI-SPEC FIX-04's
   "the h-9 row is the ONLY drag handle" invariant untouched, and avoids overcrowding a
   36px row with 5 interactive controls + the existing icon/caption/streaming-dot.
2. **`min-h-[240px]` → `min-h-[272px]`** on `GenuiPanelNode`'s outer shell (+32px, exactly
   the new toolbar row's height) so the rendered spec keeps its original ~204px content
   viewport instead of shrinking. `min-w-[320px]` unchanged. `KnowledgePreviewNode` (a
   different node type, out of this phase's scope) is untouched.
3. **Pack switcher = compact `Select`, not a `DropdownMenu` with color-swatch previews.**
   Reuses the EXACT precedent already shipped in
   `apps/web/src/app/studio/_components/generation-sandbox-island.tsx` (plain `Select` +
   `STYLE_PACKS[id].label`, no decorative color dots) — zero new visual vocabulary, zero
   new token-discipline surface area to review.
4. **Numeric bounded params use `Input type="number"` with `min`/`max`/`step`, not a
   `Slider`.** `Slider` is not currently vendored in `@polytoken/ui`
   (`packages/ui/src/*.tsx` has no `slider.tsx`). Introducing it would trigger the shadcn
   vendor-and-adapt workflow + registry safety gate mid-phase, against the
   "extend existing chrome, do not introduce a separate inspector page" mandate's spirit
   of minimal new surface area. If a future phase wants a more visual numeric control,
   that is an explicit new-vendor decision, not a default of this contract.
5. **Regenerate/re-theme in-progress = reuse the existing `<GeneratingRing>` primitive**
   (`apps/web/src/components/generating-ring.tsx`, ADOPT-03) wrapping the panel shell,
   exactly as `message-turn.tsx` already does for genui-spec streaming — instead of
   inventing a second "in-progress" visual language. This is also why it's licensed to
   consume `--primary`: 51-UI-SPEC's Accent(10%) reservation list already names "the
   generation-in-progress ring" as a valid consumer. Per `generating-ring.tsx`'s own
   contract ("must never be the sole signal"), it is always paired with an independent
   accessible signal — here, the active button's own `aria-label` swap (e.g.
   `"Regenerating panel"`) plus `motion-safe:animate-spin` on its icon.
6. **Re-theme instruction capped at 280 characters.** Not specified by 52-CONTEXT.md
   (which only says "generation-side only... one-shot resolution"). Chosen to keep the
   instruction to one concrete idea and bound downstream Bedrock cost — a plain `maxLength`
   + counter, not a hard schema requirement that needs server-side justification beyond
   normal input-length hygiene.
7. **Feedback channel split — popover-hosted vs. toast.** Actions with an open popover to
   host state (Save parameters, Apply re-theme) show errors INLINE in that popover so the
   user's input is visibly preserved (no partial apply, 52-CONTEXT.md). Actions without an
   open form (pack switch, one-click Regenerate, Restore from a list row) show
   `toast.error` with a `Retry` action, reusing the exact `{action:{label,onClick}}` toast
   shape already established in `confirm-deny-controls.tsx`. Success feedback is
   deliberately asymmetric too: routine/frequent actions (pack switch, save params,
   regenerate) get NO extra success chrome beyond the visible content change (ambient,
   never-blocking register, matching `SaveStatusIndicator`'s stated rationale); less
   frequent/more consequential state jumps (re-theme, restore) get a `toast.success` since
   the popover closing alone might not read as "your instruction was applied."
8. **TabsComponent renderer stub (999.7/52-CONTEXT.md discretion item)** is explicitly
   OUT of this UI-SPEC's scope — it is a renderer-wiring bug (Phase-19 `renderNode` never
   reached `TabsComponent`), not a visual/interaction contract question. The planner
   decides whether to fold the fix into this phase's plans per 52-CONTEXT.md's own
   discretion note; if picked up, the tab panel's own content should follow whatever
   contract governs the node type it renders (no separate spec needed here).

## Registry Safety
