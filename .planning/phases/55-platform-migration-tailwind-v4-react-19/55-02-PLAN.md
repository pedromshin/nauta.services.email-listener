<objective>
Stage 2 — the crux and highest-risk stage (55-RESEARCH.md Summary + Pitfalls 1/2/4). Port globals.css from bare-HSL-triplet CSS vars to full `oklch(...)` color functions in the shadcn v4 shape (`@theme inline` + `:root`/`.dark`), port the small JS theme surface natively to `@theme`/`@keyframes`, register the two sibling packages with explicit `@source`, remove the `@config` bridge, and — atomically in the same stage — fix every direct `hsl(var(--x))` call site so it never wraps an oklch value.

Purpose: This is the single biggest mechanical change surface in the phase, and its failures are INVISIBLE to `npm run build` (RESEARCH stresses build success ≠ migration success here): a missing `@source` silently purges vendored styling in production, and a leftover `hsl(var(--oklch-value))` is invalid CSS the browser drops silently. The oklch port and the call-site fixes MUST land together — separating them would leave surfaces rendering transparent between waves and fail the guard.

Output: globals.css in the canonical shadcn v4 oklch shape with `@source` + native `@theme` animations/radius/breakpoints; all ~16 `hsl(var(--x))` call-site files converted; the genui re-theme surface adapted; the obsolete JS configs neutralized. Verified by executable, non-visual gates.
</objective>
