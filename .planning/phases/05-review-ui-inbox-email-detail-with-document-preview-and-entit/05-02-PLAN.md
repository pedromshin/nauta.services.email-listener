<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → route handler | attachment id (untrusted path segment) crosses into the server |
| route handler → Supabase Storage | service-role-key-authenticated signed-URL request |
| route handler → browser | response body (must contain only { url } or { error }) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Information Disclosure | service-role key | mitigate | Key read from process.env server-side only; never `NEXT_PUBLIC_`-prefixed; never written to a response body (only { url }/{ error } shapes returned) |
| T-05-06 | Tampering / Path Traversal | attachment id path param | mitigate | id validated as UUID at handler entry; storage_key is looked up from the DB by id, never taken from the URL — caller cannot supply an arbitrary storage path |
| T-05-07 | Information Disclosure | error messages | mitigate | Storage/DB errors are `console.error`-logged server-side; browser receives generic friendly copy ("Failed to generate download link") per §11.5 |
| T-05-08 | Elevation of Privilege | signed URL lifetime/scope | mitigate | TTL fixed at 60s; URL is scoped to a single object in the email-attachments bucket; no write/delete grant |
| T-05-09 | Spoofing | missing secret at runtime | mitigate | Handler guards SUPABASE_URL/SUPABASE_SERVICE_ROLE_KEY presence and returns 500 with server-side log rather than constructing an unauthenticated client |
| T-05-SC | Tampering | npm install (@supabase/supabase-js) | mitigate | @supabase/supabase-js is the first-party Supabase SDK on npmjs.com (publisher: supabase) — not [ASSUMED]/[SUS]; no legitimacy checkpoint required |
</threat_model>
