<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| email_components/extraction data → DOM | untrusted email HTML body crosses into rendered DOM |
| browser → signed-URL route | attachment id crosses to the server (validated in 05-02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Tampering (XSS) | body-card HTML tab | mitigate | bodyHtml is sanitized with `DOMPurify.sanitize()` (client-only guard) before `dangerouslySetInnerHTML`; plain-text tab is the default; raw HTML is never rendered unsanitized |
| T-05-11 | Information Disclosure | error/empty copy | mitigate | Detail-load and region-load failures use the friendly §11.5 cards; the regions empty state uses non-alarming §11.4 copy (no internal error detail leaked to the user) |
| T-05-12 | Tampering | extracted_fields / labels rendered as text | mitigate | entity labels and extracted field values are rendered as text content (React auto-escapes), never via dangerouslySetInnerHTML |
| T-05-SC | Tampering | npm install (dompurify, @types/dompurify) | mitigate | dompurify is the first-party Cure53 sanitizer on npmjs.com; @types/dompurify is DefinitelyTyped — neither is [ASSUMED]/[SUS]; no legitimacy checkpoint required |
</threat_model>
