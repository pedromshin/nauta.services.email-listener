<objective>
Apply the touch-target floor (>=44px / WCAG 2.5.8) to the EXISTING
desktop-authored canvas chrome that stays visually identical for mouse users
but must grow its hit-area for any touch-capable pointer. Per 53-UI-SPEC
Judgment Call #1, this is governed by Tailwind v3.4's native `pointer-coarse:`
variant — NOT a viewport-width check — because these controls (the Phase-52
panel toolbar, the pack switcher, the knowledge-preview node buttons) only ever
render when the canvas is mounted (i.e. at >=md), so a `max-width` check would
be a no-op; a touch tablet running the canvas at >=768px is exactly the case a
width-only check silently misses.

Purpose: MOBL-02's "touch targets stay >=44px even under denser style packs" —
these are the canvas-only controls a width-based mobile check cannot reach.
Output: `pointer-coarse:` hit-area growth on six elements + a class-string test.
This is a hit-area change ONLY: glyph sizes, colors, and behavior are untouched.
</objective>
