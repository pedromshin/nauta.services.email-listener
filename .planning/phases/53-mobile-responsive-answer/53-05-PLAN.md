<objective>
Make `/chat` the market-validated inline feed below `md`: the existing docked
"chat" mode (message stream with genui panels rendered inline in flow order via
the unmodified `GenuiPartBoundary`) IS the mobile answer — and the
`dynamic(ssr:false)` React-Flow `ChatCanvasIsland` is never mounted on a phone
(not merely hidden — its init cost is never paid). The `ConversationRail`
becomes a left overlay `Sheet` below md so a fixed 280px rail can't squeeze the
chat column to ~80px at 360px.

Purpose: MOBL-01 — "chat canvas collapses to an inline-first feed below md;
desktop keeps the 2D canvas." Zero new feed-item components: reuse
MessageList/MessageTurn/Composer verbatim (53-UI-SPEC Component Inventory §2).
Output: `page.tsx` gates the canvas on `useIsMobileViewport()`; `conversation-
rail.tsx` renders a Sheet overlay below md; a vitest suite proves the island
never mounts on mobile even when canvas mode is forced.
</objective>
