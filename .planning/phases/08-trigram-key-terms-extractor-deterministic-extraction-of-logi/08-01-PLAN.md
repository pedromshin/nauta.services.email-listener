<objective>
Activate the dormant pg_trgm arm of hybrid RRF retrieval and fix the confirm-fallback
NOT NULL FK bug. Two deliverables, both backend-only Python (apps/email-listener):

1. A pure domain service `extract_key_terms(text) -> tuple[str, ...]` that deterministically
   pulls logistics identifiers (ISO 6346 container numbers with check-digit validation,
   label-anchored BL / booking / PO / invoice numbers) out of region text, then wire it
   into `AutofillUseCase` so retrieval receives real key_terms instead of `key_terms=()`.
2. Fix `ConfirmRegionUseCase`'s no-candidate fallback so it never writes
   `ExtractionRecord(entity_type_id="")` (which violates the live NOT NULL UUID FK) —
   skip-and-warn, while still embedding + indexing the component (D-15 flywheel still closes).

Purpose: The trgm RPC + GIN index have been live since migration 0009 but the arm is inert
because the query never supplies key_terms. This makes identifier-based few-shot retrieval
work, and removes a live crash on the confirm fallback path.

Output: One new domain service + its TDD test file, edits to two use cases, edits to two
unit test files, and an extended real-Postgres integration test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| email/PDF text → extractor | Untrusted document text (megabytes possible) is scanned by regexes in key_terms.py |
| extractor terms → trgm RPC | Extracted identifier strings flow into match_components_by_trgm as parameters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Denial of Service | `extract_key_terms` regexes | mitigate | Linear-time patterns only — bounded repetition (`{m,n}`), no nested quantifiers, no catastrophic backtracking; cap scanned input via `_MAX_SCAN_CHARS` (slice before scanning); cap output via `_MAX_TERMS`. Verified by the large-input test in Task 1. |
| T-08-02 | Tampering / Injection | extracted terms → `match_components_by_trgm` RPC | accept | Terms are returned as data and passed as a bound parameter to the already-parameterized RPC (04-08 RetrievalPort contract); no string interpolation into SQL anywhere in this phase. No new injection surface introduced. |
| T-08-03 | Information disclosure | confirm-fallback warning log | accept | `confirm_region_no_candidate_record_skipped` logs only component/importer ids already in the bound logger context; no document content or PII added. |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies — extractor is stdlib `re` only; no package installs in this phase, so no legitimacy gate required. |
</threat_model>
