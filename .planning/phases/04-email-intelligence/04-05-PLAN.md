<objective>
Add the LLM segmentation pass (D-08): given a parsed page's content (text-layer text or OCR words with boxes), Claude proposes candidate entity regions — supporting multiple, overlapping, and nested entities on one page, many-pages-as-one-entity, and junk/corrupt content. Proposals become child Components (nested via parent_component_id) that a future UI can accept/redraw/split/merge/reject (D-09). The Claude call is structured so untrusted content stays in the user turn inside delimiters (D-14).

Purpose: This is the intelligence that turns flat pages into addressable entity regions — the robustness bar from the "chaotic single page" specific.
Output: Anthropic client wrapper, SegmenterProtocol + AnthropicSegmenter, ProposeRegionsUseCase, container wiring, with tests for the multi/overlapping/nested/junk cases and the prompt-injection structural defense.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| document content → Claude | Untrusted email/attachment text crosses into an LLM call — the prompt-injection surface (design case #6) |
| Anthropic API → app | LLM output is untrusted structured data that gates no action directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-15 | Tampering | prompt injection via document content | mitigate | D-14 structural defense: content lives only in the user turn inside <document_content> delimiters; system prompt is a constant carrying schema/instructions only. Injection test asserts system is byte-identical regardless of content (RESEARCH §6 points 1+3). |
| T-04-16 | Elevation of Privilege | LLM "instruction" in document triggers an action | mitigate | Segmentation output is parsed as structured JSON region proposals only; nothing in the response drives a side effect beyond inserting candidate rows. Actions gate on deterministic code, never on model free-text (RESEARCH §6 point 3). |
| T-04-17 | Denial of Service | giant page text inflates token cost / hangs | mitigate | _generate has bounded retries; cap page_text length sent to the model (truncate at a token budget) and log truncation. |
| T-04-18 | Information Disclosure | document content sent to Claude via AWS Bedrock | accept | Claude-on-Bedrock is the chosen extraction LLM (CONTEXT integration points); invoked via the ECS task IAM role (bedrock:InvokeModel), so content stays within the AWS account boundary — no third-party API key, no cross-account egress. Documented in-AWS processing boundary. |
</threat_model>
