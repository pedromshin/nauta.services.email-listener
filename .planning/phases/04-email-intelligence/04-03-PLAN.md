<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AWS Secrets Manager → ECS task env | Supabase secret key + Anthropic key are injected by ECS from Secrets Manager into the task environment; the app reads them as env vars and must never log or commit them |
| app code → Postgres (service_role) | the secret API key bypasses RLS, so importer_id filtering in code is the multi-tenant enforcement layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Information Disclosure | Cross-tenant read because the secret key bypasses RLS | mitigate | Every repository query includes `.eq("importer_id", ...)`; reads not directly importer-scoped are reached only via email_id/attachment_id that are themselves tenant-bound. Tests assert the filter is present. |
| T-04-08 | Information Disclosure | Secret key logged or surfaced in error | mitigate | Secrets read only via parse_secret_value; client.py raises a generic RuntimeError naming the env var, never the value; bandit runs in verify. |
| T-04-09 | Elevation of Privilege | Missing secret in prod silently uses empty key | mitigate | get_supabase_client raises RuntimeError on empty URL/key — fail closed at startup (D-04). |
| T-04-03 | Elevation of Privilege | SUPABASE_SECRET_KEY (new-format sb_secret_…) exposure grants full DB | mitigate | The key is injected into the ECS task environment by ECS from AWS Secrets Manager ({env}/nauta-services/SUPABASE_SECRET_KEY) — not fetched by app code at startup and never committed; IAM execution role scopes GetSecretValue to the specific secret arns. ANTHROPIC_API_KEY follows the identical injection pattern. |
| T-04-10 | Tampering | Reprocess overwrites a human-confirmed value | mitigate | supersede_active never deletes and never touches confirmed rows' field data; it only flips status to superseded, preserving audit history (D-16). |
| T-04-SC | Tampering | new packages supabase / anthropic from PyPI (supply chain) | mitigate | See checkpoint in verification — confirm both on pypi.org before `uv sync` lands them. |
</threat_model>
