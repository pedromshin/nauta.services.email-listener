<objective>
Define the Clean-Architecture domain layer for Email Intelligence: frozen dataclass entities (Email, Attachment, Component, EntityType+EntityTypeField, ExtractionRecord) mirroring the Supabase schema, repository Protocol ports, the format-agnostic ParserProtocol seam (D-10), and the ParserRegistryPort callable type that lets the application layer dispatch parsers without importing infrastructure. This is the contract layer every later plan implements against — handing executors blueprints, not "build me a house."

Purpose: Lock the interfaces first so PDF parsing (04-04), segmentation (04-05), repositories (04-03), autofill (04-07), and retrieval (04-08) all build against stable types with no codebase scavenger hunt.
Output: Domain entities + ports + ParserProtocol + ParserRegistryPort, all import-linter-clean, with unit tests proving immutability and the parser seam contract.
</objective>
