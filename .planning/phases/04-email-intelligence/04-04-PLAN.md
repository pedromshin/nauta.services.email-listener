<objective>
Build the PDF parser to full production quality — the head of the file-type distribution, no stubs (D-06). Per-page adaptive extraction (D-07): detect a usable text layer (pdfminer/pypdf) and use it; when absent or garbage (scanned, photographed, photo-of-screen), fall back to AWS Textract OCR. Every page becomes a Component carrying normalized 0–1 geometry (D-12), and the whole thing registers behind the ParserProtocol seam (D-10). Corrupt/encrypted/junk PDFs degrade gracefully.

Purpose: This is the depth-first deliverable — the one format built for real. All three PDF realities from the design case [C1] must work end-to-end.
Output: parser registry, PdfParser, text-layer detector, OCRProtocol + Textract adapter, with contract + integration tests over real fixtures.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| attachment bytes → parser | Untrusted, possibly adversarial PDF (zip-bomb, malformed, encrypted, photo-of-hostile-image) crosses into the worker |
| parser → AWS Textract | image bytes sent to a third-party OCR service |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Denial of Service | malformed/zip-bomb/huge PDF exhausts memory or hangs | mitigate | Page work runs in a bounded ThreadPoolExecutor; cap page count and per-page raster size (MAX_PAGES, DPI limit) and reject oversize file_bytes before processing; per-page try/except prevents a single bad page from killing the run. |
| T-04-12 | Tampering | adversarial OCR text later reaches the LLM | mitigate (here) | Parser emits content_text only as data; the prompt-injection defense is structural and enforced in 04-05/04-07 (content stays in the user turn, D-14). Parser never executes or interprets extracted text. |
| T-04-13 | Denial of Service | encrypted/corrupt PDF raises and crashes the decompose worker | mitigate | Whole-document and per-page failures degrade to a parse-error Component with a logged warning; parse() never propagates an exception for content problems. |
| T-04-14 | Information Disclosure | document bytes sent to Textract leak sensitive content | accept | Textract is the chosen AWS-native OCR (RESEARCH §12 Q1), same trust domain as the rest of the stack; region pinned via AWS_TEXTRACT_REGION. Documented as an accepted third-party processing boundary. |
| T-04-SC | Tampering | new PyPI deps (pypdf, pdfminer.six, pdf2image, boto3, pillow) | mitigate | All are well-known first-party/established packages; confirm on pypi.org before `uv sync` (see verification). |
</threat_model>
