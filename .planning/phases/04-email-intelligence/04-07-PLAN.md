<objective>
Implement cold-start autofill (D-13): when a human registers a region and clicks autofill with zero prior confirmed examples, the LLM extracts field values from the region content + the entity type's preset description + default knowledge base + field schema (no retrieval at cold start). The result is a candidate ExtractionRecord. Content is structurally isolated to the user turn (D-14). Expose it as POST /v1/components/{id}/autofill.

Purpose: This is the "click-to-autofill" half of the loop — the first email matches against nothing, so it leans on type defaults. It also defines the prompt shape the retrieval plan (04-08) extends with few-shot examples.
Output: AutofillProtocol + AnthropicAutofiller, AutofillUseCase, components API, container/app wiring, with tests for cold-start behavior and the prompt-injection defense.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| region content → Claude | Untrusted extracted content crosses into the extraction LLM |
| client → /v1/components/{id}/autofill | Authenticated; component_id + entity_type_slug are attacker-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-24 | Tampering | prompt injection via region content steering extraction | mitigate | D-14: region content only in the user turn inside <document_content>; system prompt carries schema/type/KB only; injection test asserts system byte-identical. Knowledge base text fed to the system is type-default config, NOT derived from email content (RESEARCH §6 point 2). |
| T-04-25 | Elevation of Privilege | autofill output auto-writes somewhere | mitigate | AutofillUseCase only inserts a status=candidate ExtractionRecord; nothing auto-confirms or writes to Nauta this phase (everything human-reviewed — CONTEXT scope-out). |
| T-04-26 | Information Disclosure | cross-tenant component/entity-type read | mitigate | find_by_id/find_by_slug are importer-scoped; the use-case requires importer_id; system-default entity types (importer_id NULL) are intentionally shared. |
</threat_model>
