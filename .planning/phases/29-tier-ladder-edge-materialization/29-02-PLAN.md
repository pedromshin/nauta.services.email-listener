<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OCR-derived text/tokens → DB write | region token text (untrusted document content) enters the provenance jsonb and node content |
| edge → tenant scope | edges have no importer_id; tenant isolation holds only via source_node_id → knowledge_nodes.importer_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-04 | Injection | provenance jsonb / node content | mitigate | All OCR-derived strings and token payloads pass through `strip_nul` before write (existing sanitize idiom); values are bound via the PostgREST client (parameterized), never string-concatenated into SQL |
| T-29-05 | Tampering | supersede path | mitigate | deactivate_edges_for_node uses `.update(is_active=false)` only; `.delete()` is asserted never-called in tests — audit trail preserved |
| T-29-06 | Elevation of privilege | cross-tenant edge write | mitigate | upsert_node always sets importer_id from the caller-supplied tenant; the synthesizer (29-03) only ever creates edges whose source_node_id belongs to a node it just wrote for that importer_id — no path to attach an edge under another tenant's node |
</threat_model>
