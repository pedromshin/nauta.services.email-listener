<objective>
Deliver the canvas surface and the view toggle (CANVAS-01, D-01/D-02/D-03). First, lift Phase 22's streaming/turn state out of `ConversationView` into a `useConversationController` hook so the docked Chat view and the new canvas ChatNode read ONE shared controller — switching views never interrupts an in-flight generation (D-02). Then build the React Flow surface: a `ChatNode` (full composer, reusing MessageList/Composer), the genui-panel nodes materialized from each `genui_spec` turn (dagre auto-placed, D-03), a module-level `nodeTypes` map, `dynamic(ssr:false)` island, empty/skeleton/keyboard-hint chrome, and full drag/pan/zoom + keyboard operability — all mounted behind a `ChatCanvasViewToggle` (Tabs, localStorage per conversation).

Purpose: This turns the contracts from 23-01/23-02 into a visible, interactive canvas. Persistence (exact restore + debounced save) and streaming responsiveness land in 23-04; shared-state/data-edges in 23-05.

Output: `useConversationController`, refactored docked view, `ChatNode`, `nodeTypes`, `canvas-layout.ts` (dagre port), `ChatCanvas` + island + skeleton + empty state + view toggle + keyboard hint.
</objective>
