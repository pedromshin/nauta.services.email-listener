<objective>
Deliver the shared per-chat state store and data-carrying edges (STATE-01, STATE-02, D-08/D-09/D-10, FOUND-4). A per-conversation Zustand store implements a SUPERSET of the v1.1 declared-state model — the SAME bounded 5-mutation enum (toggle/set/reset/increment/decrement, no arbitrary reducers), the SAME dotted-path binding grammar (`resolveDataRef` + FORBIDDEN_KEYS), with `panels.{panelId}.{key}` and `shared.*` namespaces. Data-carrying edges declare `{ sourcePath, targetKey }`: panel B's `data` receives panel A's output value at `targetKey` via the store (a live subscription that re-resolves on source change). Edge payloads are Zod-validated and guarded exactly like declared-state; `EdgeCreationPicker` requires explicit confirmation (never auto-fires). Store contents + edges persist in the `chat_canvas_layouts` row and survive reload. `SpecRenderer` stays UNMODIFIED — panels receive store values through the existing `data` prop.

Purpose: This completes the phase — panels stop being isolated and start sharing state and data, on the FOUND-4 "one state system, never two" posture.

Output: `canvas-store` (+ context), `EdgePayloadSchema`, `DataEdge`, `EdgeCreationPicker`, store→panel-data wiring, and shared_state persistence — behind a blocking zustand legitimacy checkpoint.
</objective>
