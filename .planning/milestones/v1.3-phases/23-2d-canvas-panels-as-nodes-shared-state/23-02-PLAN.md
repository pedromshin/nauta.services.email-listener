Content-hash registry pattern to mirror (packages/genui/src/registry/registry-version.ts):
```typescript
import { createHash } from "crypto";
// sorts entry keys, JSON.stringify a stable public surface per entry, sha256 hex
export const REGISTRY_VERSION = { catalogId: "global", version: computeRegistryHash(COMPONENT_REGISTRY) };
```
NOTE: `crypto.createHash` is Node-only. A client-imported registry version must be a build-time constant or use a browser-safe hash; compute the hash over a STABLE STRING (sorted type ids + each entry's Zod schema `.toString()`/description) and, to stay client-safe, use a small pure hash (e.g. a deterministic FNV-1a/djb2 over the stable string) rather than Node `crypto`. The property that matters (D-04): ANY registry entry change flips the version.
