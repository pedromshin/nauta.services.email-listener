<objective>
Wire thread grouping into the live ingest pipeline (THRD-01): a `ThreadResolver` domain port (mirroring `ImporterResolver`) resolved at ingest time, a Supabase adapter that finds the incoming email's thread neighbors within its importer and returns a find-or-created/merged `thread_id`, the `thread_id` threaded onto the `Email` entity + repository, and a re-runnable idempotent backfill that assigns every existing email a thread.

Purpose: Turn the pure grouping algorithm (Plan 45-02) into persisted threads so the inbox (Plan 45-04) can group by them and E3 (next epoch) can consume the model. Same-importer scoping keeps grouping tenant-safe.
Output: ThreadResolver port + Supabase impl, ingest wiring, Email.thread_id, and an executed backfill.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inbound email → ThreadResolver | attacker-controlled headers/body drive grouping; must stay importer-scoped |
| backfill script → DB | bulk writer; must not cross importer boundaries or corrupt existing threads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-03-01 | Spoofing | forged References/embedded ids to merge into another tenant's thread | mitigate | every resolve()/backfill query filters `.eq("importer_id", importer_id)` — cross-importer emails are never candidates; importers are user-scoped (Phase 44) so cross-tenant merge is impossible |
| T-45-03-02 | Tampering | resolver exception aborts ingestion / drops mail | mitigate | resolution is best-effort — exception → thread_id=None + logged warning, ingestion continues (mirrors propose_regions isolation); SNS handler already returns 200 |
| T-45-03-03 | Tampering | backfill re-run corrupts/duplicates threads | mitigate | deterministic group_emails + reuse-canonical rule → idempotent; `--dry-run` verifies before writing |
| T-45-03-SC | Tampering | dependency install | accept | No new packages; reuses existing supabase Client + stdlib |
</threat_model>
