Add a minimal, additive, NON-enforcing FastAPI reader that lands `X-User-Id` into request context for Phase 44 to enforce — without touching `require_api_key` and keeping all existing service tests green.

Purpose: AUTH-04. FastAPI is reachable only server-to-server through the authenticated BFF; `X-API-Key` still means "from our trusted Next.js server", the new `X-User-Id` means "acting on behalf of this user". FastAPI ENFORCEMENT of that identity is deliberately Phase 44 scope (accepted here).
Output: `X-User-Id` forwarding in the 4 FastAPI-proxying routes + a non-enforcing `user_context.py` extractor with tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → BFF proxy route | Client cannot be trusted to state its own identity |
| BFF proxy → FastAPI (service-to-service) | X-API-Key proves "from our server"; X-User-Id must be server-derived to mean anything |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-P4-01 | Spoofing (client-supplied identity) | BFF proxy X-User-Id | mitigate | `X-User-Id` is derived from `getUser()` server-side; routes never read an inbound user id for the forwarded value (Pitfall 9) |
| T-43-P4-02 | Tampering (service-boundary regression) | `require_api_key` / X-API-Key | mitigate | X-API-Key header + `require_api_key` left byte-for-byte unchanged; extractor is additive; existing pytest suite stays green |
| T-43-P4-03 | Spoofing (unverified session) | proxy user resolution | mitigate | Identity resolved via server-verified `getUser()`, never `getSession()` |
| T-43-P4-04 | Elevation (FastAPI over-trusting forwarded id) | FastAPI extractor | accept | Non-enforcing by design; enforcement is Phase 44 scope (owner: Phase 44 tenancy sweep) — documented in the extractor docstring |
| T-43-P4-05 | Information Disclosure (identity in logs) | X-User-Id value | mitigate | Value is an opaque user UUID, not a secret; not logged alongside API keys |
</threat_model>
