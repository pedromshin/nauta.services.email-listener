# Nauta Test Corpus

A layered PDF test corpus for the Phase 4 email-intelligence pipeline (D-17).
The corpus proves the `PdfParser → ProposeRegionsUseCase` stack against authentic
document messiness before the phase is considered complete.

---

## Layer Structure

```
tests/corpus/
  scan_noise/        # Image-only PDFs: scan noise (RVL-CDIP, DocLayNet provenance)
  logistics_vocab/   # Hand-authored logistics templates: BL, invoice, packing list, booking
  hard_cases/        # Controlled hard cases with ground truth (D-17)
  manifest.json      # Per-file metadata (layer, provenance, expected_entity_types)
  ground_truth.json  # Expected entity types + key identifiers for hard_cases/
  forwarding_harness.py   # Async helper to feed a corpus file through IngestInboundEmailUseCase
```

---

## Layer 1 — scan_noise/

Synthetic image-only PDFs representing the kind of scan noise found in real
logistics documents.

| File | Provenance |
|------|-----------|
| `rvl_cdip_invoice_001.pdf` | Modelled after the RVL-CDIP invoice class. Full dataset: https://www.cs.cmu.edu/~aharley/rvl-cdip/ — RVL-CDIP terms (research use). Committed file is a programmatically-generated JPEG-in-PDF for CI reproducibility. |
| `doclaynet_invoice_001.pdf` | Modelled after DocLayNet invoice pages. Full dataset: https://github.com/DS4SD/DocLayNet — Apache 2.0. Committed file is programmatically generated. |

**Purpose:** Verify PdfParser routes image-only pages through the OCR path
(no text layer detected). Real OCR results require live AWS Textract credentials
and are gated with `@pytest.mark.integration`.

**Regeneration:** These files are generated by `gen_corpus.py` at the repo root.
Run `uv run python gen_corpus.py` from the project root to regenerate.

## Layer 2 — logistics_vocab/

Hand-authored format templates with realistic logistics vocabulary and
identifiers. All parties are fictional; container numbers follow ISO 6346 format.

| File | Document Type | Key Identifiers |
|------|--------------|----------------|
| `bill_of_lading_template.pdf` | Bill of Lading | BL: HLCUSHA230901234, Container: MSCU1234567 |
| `commercial_invoice_template.pdf` | Commercial Invoice | Invoice: INV-2024-00892, Booking: BKG-2024-44521 |
| `packing_list_template.pdf` | Packing List | Ref: PKL-2024-00892, Container: MSCU1234567 |
| `booking_confirmation_template.pdf` | Booking Confirmation | Booking: BKG-2024-44521, Container: MSCU1234567 |

**Purpose:** Validate that the pipeline extracts correct entity types and
key identifiers from standard logistics formats.

**Regeneration:** `uv run python gen_corpus.py` from project root.

---

## Layer 3 — hard_cases/ (D-17)

Four controlled hard cases with paired ground truth in `ground_truth.json`.

| File | Hard Case | D-17 Criterion |
|------|-----------|---------------|
| `multi-invoice-in-one-pdf.pdf` | Two complete invoices on separate pages | ≥ 2 invoice-typed proposed regions |
| `nested-entities-on-one-page.pdf` | BL + container status table + invoice ref on one page | Region children carry `parent_component_id` linkage |
| `junk-corrupt.pdf` | Truncated/corrupt PDF with garbage trailer | Parser must not raise; 0 or only unknown regions |
| `photo-of-screen.pdf` | Image-only PDF (photographed screen, no text layer) | No text layer detected; routed through OCR |

## ground_truth.json Schema

```json
{
  "<relative-file-path>": {
    "description": "human-readable description",
    "expected": {
      "min_region_count": 0,
      "entity_types": ["invoice", "bill_of_lading", ...],
      "key_identifiers": { "<field>": "<value>" },
      "assertions": {
        "<assertion-name>": "<description of what must hold>"
      }
    }
  }
}
```

Keys are paths relative to `tests/corpus/`. The `expected.assertions` object
is human-readable documentation of what the test `test_corpus_pipeline.py`
asserts for each hard case.

## Forwarding Harness

`forwarding_harness.py` exports:

```python
async def forward_corpus_file(
    path: Path,
    *,
    importer_id: str,
    content_type: str = "application/pdf",
    fake_segmenter: SegmenterProtocol,
) -> tuple[Email, list[Component]]:
    ...
```

Wraps the corpus file as a minimal RFC 2822 MIME message with one PDF
attachment and drives `IngestInboundEmailUseCase.execute()` using in-memory
fakes for all infrastructure ports (no S3, no Supabase, no AWS credentials).

Returns `(persisted_email, all_components)` including both attachment_page
Components (PdfParser output) and region Components (ProposeRegionsUseCase
output).
